## 🔬 RQ1: 온보딩부터 등록까지의 전체 플로우를 별도 안내 없이 자연스럽게 완수할 수 있는가?

> 각 단계에서 시스템이 자동 처리한 것과 셀러가 직접 판단해야 하는 것의 경계를 혼동 없이 구분할 수 있는지 포함하여 관찰한다.
> 

### 판정: ⚠️ 두 세션 모두 조건부 완수 — 경력 셀러 기준, 총 5개 인지 혼란 지점 도출

두 세션 모두 전체 플로우를 완수했으나, **"경력 셀러이기 때문에"** 완수 가능한 수준이었음. 신규 셀러 대상에서는 이탈 가능성이 높음.

| **혼란 지점** | **1차 (손건우 주임님)** | **2차 (홍찬열 팀장님)** | **통합 판단** |
| --- | --- | --- | --- |
| 자동수집 동작 방식 인지 | 자동수집 원해서 자동수집 탭 진입 → 진입 자체는 맞음. 그러나 설정 완료 후 오전 7시 예약 실행임을 인지 못함 (CO-1) | 자동수집 설정 완료 후 즉시 수집 시작으로 오해 (2차) | **양 세션 공통** — 스케줄 기반 동작 방식을 UI에서 명시 필요 (P1) |
| 자동화 경계 불명확 | 자동 채운 값 vs 수동 입력값 구분 안 됨 (ON-3) | 자동수집 설정 후 즉시 수집으로 오해. "글 안 읽게 됨" (2차) | **양 세션 공통** — 시스템 자동화 범위를 텍스트가 아닌 UI 요소로 표시 필요 (P1) |
| AI 기능 기대-현실 갭 | "AI 편집 = 번역 자동화" 기대 (ED-1) | AI 상세 설명 빈칸 → 어디서 오는 정보인지 불명확 (2차) | **양 세션 공통** — AI 기능 범위를 진입 전에 명시 / 출처 표시 (P1) |
| 기술 버그 | 윈도우 텍스트 깨짐 (ON-4) | 탭 이동 버그로 플로우 물리적 중단 (2차 P0) | 각각 별도 수정 필요. 2차 탭 버그는 출시 전 P0 |

<aside>
🔴

**통합 의사결정 A: "자동화 경계 시각화" 패턴을 전체 플로우에 일관 적용 (P0~P1)**

두 세션에서 반복 확인된 가장 강력한 신호. 자동 수집 완료 상태, 자동 채운 값, AI 작성 영역 모두 텍스트 설명 대신 상태 배지·아이콘·토스트 등 시각적 UI 요소로 전달해야 함.

- **1차 근거**: ON-3 (자동/수동 입력값 경계 불명확) / ED-1 (AI 번역 자동화 기대갭)
- **2차 근거**: 자동수집 "예약 완료" 상태 피드백 부재 / AI 상세설명 빈칸 혼란
</aside>

<aside>
🟡

**통합 의사결정 B: 자동수집 동작 방식(스케줄 기반) UI 명시화 (P1, 1차+2차 공통)**

참가자는 자동수집을 원했고 자동수집으로 진입한 것 자체는 맞는 선택이었음. 핵심 문제는 설정 완료 후 "지금 바로" 수집이 아닌 "매일 오전 7시"에 예약 실행된다는 사실이 화면 어디에도 드러나지 않아 혼란이 발생한 것.

- **1차 근거 (CO-1)**: 설정 저장 후 수집이 즉시 시작되지 않아 설정 완료 여부 확인 불가. 탭 라벨 또는 설정 완료 시 "다음 수집: 내일 오전 7시" 예약 확인 피드백 필요.
- **2차**: 동일하게 자동수집 설정 완료 후 즉시 수집 시작으로 오해 → 양 세션 공통 구조적 문제로 확인됨
</aside>

### 공통으로 문제 없었던 구간

- **편집 → 등록 전환**: 두 세션 모두 별도 안내 없이 자연스럽게 완수
- **변동 알림 설정**: 두 세션 모두 기능 목적 즉시 이해 ("리스팅 놓칠 수 있는 것 방지")
- **가격 설정 구조**: 두 세션 모두 마진율/수익 계산 구조에 큰 마찰 없음

## 🔎 RQ2: 각 화면에 표시된 정보 중 셀러가 불필요하다고 느끼거나, 반대로 부족하다고 느끼는 정보는 무엇인가?

### 판정: ❌ 두 세션 모두 부족 우세 — "정보 과다" 화면은 단 하나도 없었음

| **부족/불명확 정보** | **1차** | **2차** | **우선순위** |
| --- | --- | --- | --- |
| 배송대행사 ↔ 요율표 연동 로직 | "배송대행이랑 해외배송 요율표는 다른 개념인가요?" (ON-2) | 배송비 "직접 세팅" 옵션 요구 (2차) | P1 🟡 — 양 세션 공통 |
| 원본 정보 ↔ 업로드 예정 정보 분리 | "수집된 정보랑 Q10 업로드 될 정보 확인 원함" (ED-2, ED-3) | "상세 설명 출처가 어딘지 설명이 없어" (2차) | P1 🟡 — 양 세션 공통 |
| 자동저장 상태 표시 | "탭 옮기면 사라지는 프로그램 있었어서" (ED-4) | 직접 언급 없음 | P1 🟡 — 1차 확인 |
| 소싱 링크 바로가기 | "소싱 링크 바로 알 수 있는 링크 하나 있으면" (NT-3) | 직접 언급 없음 | P1 🟡 — 1차 확인 |
| 자동수집 예약 완료 상태 피드백 | 직접 언급 없음 | 즉시 수집 시작으로 오해 (2차) | P1 🟡 — 2차 확인 |
| AI 상세 설명 출처 및 CTA | AI = 번역 자동화 기대 (ED-1) | "빈칸이었다. 어디서 긁어오는지 설명이 없어" (2차) | P1 🟡 — 양 세션 공통 (표현만 다름) |

<aside>
🔴

**통합 의사결정 C: "원본 vs 업로드 예정" 데이터 레이어 분리를 핵심 UX 원칙으로 확정 (P1)**

두 세션 모두 소싱 원본 정보와 편집/업로드 예정 정보의 경계를 인식하지 못해 혼란이 발생함. 단순 UI 개선이 아니라 **데이터 구조 설계 원칙**으로 전사 적용 필요.

- **1차**: "원본 상품명 + 올릴 상품명 칸을 따로" 직접 제안 (ED-2)
- **2차**: AI 작성 정보의 출처와 자동화 범위가 불명확 (2차)
</aside>

<aside>
🟢

**통합 의사결정 D: 등록 기록 기능 MVP에서 제외 권고 (P2)**

유일하게 "불필요"로 지목된 기능. 개발 리소스를 변동 알림 플로우 완성에 집중하는 것이 효과적.

- **2차 근거**: 참가자가 직접 불필요하다고 언급
- **1차**: 별도 언급 없음 (무관심 = 중요도 낮음)
</aside>

### 두 세션 공통으로 정보가 충분했던 영역

- **온보딩 설정 항목 전반**: 경력 셀러 기준 "필요한 정보가 다 있다"
- **가격/마진율 구조**: 두 세션 모두 마찰 없이 수용
- **수집 리스트 화면**: 상품 정보 인식에 큰 어려움 없음

## 🏆 RQ3: 기존에 사용하던 솔루션(셀러픽, 투플렉스) 대비 체감되는 차별적 가치가 있는가?

### 판정: ✅ 두 세션 모두 차별적 가치 확인 — 핵심 차별점 3개 교차 검증 완료

두 참가자가 서로 다른 경쟁사(2Flex vs 셀러픽)를 기준으로 평가했음에도 **공통된 차별점 3개**가 도출됨 → 신호의 신뢰도가 높음.

### 🥇 차별점 1위 (양 세션 공통): 변동 알림 — Pain Point 직접 해소

<aside>
🔥

**통합 의사결정 E: 변동 알림을 핵심 포지셔닝 1순위로 확정, 감지→알림→자동 대응 파이프라인을 2차 MVP로 편성 (P1)**

- **1차 근거 (CL-1, NT-1, NT-4)**: "자동 시스템이 좋을 것 같고 변동 알림도 괜찮다" — 2Flex에는 없는 기능
- **2차 근거**: 마무리에서 "가장 먼저 개선됐으면 하는 기능"으로 유일하게 지목. "역마진/품절 시 자동 판매중지가 되면 훨씬 유용하다"
- **공통 한계**: 두 세션 모두 "알림→수정→재업로드" 플로우가 끊어진다고 지적. 알림만 있고 인앱 액션이 없으면 DayZero는 "중간에 끊어지는 허브"가 됨
- **공통 확장 방향**: 감지 → 알림 → 자동 대응(일시정지, 가격 재설정)까지 연결해야 진정한 차별점
</aside>

### 🥈 차별점 2위 (양 세션 공통): AI 기능 (번역 + 일괄편집)

<aside>
🔥

**통합 의사결정 F: AI 기능을 2순위 차별점으로 포지셔닝, 대량 처리 시나리오 체험 유도 필수 (P1)**

- **1차 근거 (ED-5, CL-3)**: 초반 "큰 차이 없다" → 세션 말미 "AI 일괄편집 기능이 좋다. 감동이 될 수 있을 것 같다" — 단건 시나리오에서는 차별점 미체감, 대량 처리 시나리오에서 가치 급상승
- **2차 근거**: "셀러픽에는 번역 기능 없어서 ChatGPT·파파고 별도 사용 중" — AI 번역이 기존 3단계(복사→외부 AI→붙여넣기) 워크플로우를 1단계로 축소
- **공통 전략**: 온보딩 또는 편집 단계에서 **20개+ 대량 처리 시나리오를 먼저 체험**시켜야 가치 인식이 빠르게 높아짐
</aside>

### 🥉 차별점 3위 (2차 확인): 수집 정보 완결성 (카테고리 자동 매핑 + 무게 정보)

<aside>
🔥

**통합 의사결정 G: 카테고리 자동 매핑 + 무게 정보를 셀러픽 비교 포인트로 마케팅 활용 (P2)**

- **2차 근거**: "카테고리 불러와지는 게 좋다" / "상품 페이지에서 무게 가져올 수 있는 것" — 셀러픽 대비 명확한 정보 완결성 우위
- **1차**: 직접 언급 없음 (2Flex와 비교하는 프레임이라 다른 지점에 집중)
</aside>

### ⚠️ 두 세션 공통 경쟁사 우위 영역 (인정해야 할 부분)

<aside>
⚠️

**통합 의사결정 H: 반자동 수집(크롬 익스텐션)을 2차 MVP 로드맵에 편성, 단기는 "양보다 질" 포지셔닝 (P2)**

- **1차 근거 (CO-6, CL-4)**: 2Flex 확장프로그램이 클릭 수집 UX의 기준점. 기존 등록 상품 일괄 수정(마진율·이미지) 기능도 2Flex 강점
- **2차 근거**: 셀러픽의 크롬 익스텐션 반자동 방식이 "빠르게 많이" 수집하는 데 더 편함
- **공통 대응 전략**: 반자동 미구현 시 수집 단계 전환 유인 부족. 단기적으로는 수집 정보 완결성(카테고리·무게·가격)을 "양보다 질" 포지셔닝으로 방어
</aside>
