## 화면별 명세 구조

모든 화면 명세는 동일한 템플릿을 따릅니다:

```
# [화면ID] 화면명

## 화면 목적
이 화면이 존재하는 이유 한 줄.

## 진입 경로
어떤 경로로 이 화면에 도달하는가.

## 레이아웃
텍스트 와이어프레임으로 영역 구성 표시.

## 상태별 분기
빈 상태 / 정상 / 로딩 / 에러 등 조건별 화면 변화.

## 동작 정의 (Given/When/Then)
사용자 행동 → 시스템 반응을 BDD 형식으로 정의.

## UI 텍스트
화면에 표시되는 모든 텍스트와 표시 조건.

## 이동 경로
이 화면에서 어디로 갈 수 있는가.

## 검증 체크리스트
구현 후 확인할 항목 목록.
```

## 핵심 원칙

- **이 문서에 없는 UI 요소는 구현하지 않는다**
- **이 문서의 UI 텍스트를 그대로 사용한다** (LLM이 임의 생성하지 않는다)
- **검증 체크리스트를 구현 완료 기준으로 사용한다**
- **불분명한 부분은 코드를 먼저 확인하고, 그래도 불분명하면 질문한다**
