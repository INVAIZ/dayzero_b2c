# 유저 플로우

> 사용자가 실제로 경험하는 주요 시나리오별 여정을 정의합니다.
> 각 단계에서 어떤 화면을 거치고, 어떤 조건에서 분기가 발생하는지 명시합니다.
> 화면별 세부 동작은 `screens/` 개별 명세를 참조하세요.

---

## 플로우 1: 신규 가입 → 첫 상품 등록 완료

전체 파이프라인을 처음부터 끝까지 경험하는 핵심 여정.

```
S-01 로그인
 │ 회원가입 (이메일 또는 Google)
 ▼
S-02 Qoo10 연동
 │ API Key 입력 → 연동 성공
 ▼
S-03 기본 정보
 │ 배송대행사 선택 → 주소 자동입력 → 연락처 → 배송비 → 마진율 설정
 ▼
S-05 익스텐션 설치
 │ 설치 또는 건너뛰기
 ▼
SC-01 수집 홈 ← 메인 진입점
 │ 자동 수집 또는 직접 수집 선택
 ▼
┌─────────────────┬──────────────────┐
│ 자동 수집        │ 직접 수집         │
│                 │                  │
│ SC-03 설정      │ SC-02 URL 입력   │
│  쇼핑몰 선택     │  URL 붙여넣기     │
│  카테고리 선택   │  수집 시작        │
│  시간 설정       │  순차 처리        │
│  저장            │  완료             │
│                 │                  │
│ → SC-01 대기    │ → ED-01 이동     │
│ (다음날 아침     │                  │
│  수집 실행)      │                  │
└─────────────────┴──────────────────┘
 │ 수집 완료 (자동 수집: 알림으로 확인)
 ▼
ED-01 편집 목록
 │ 상품 선택 → AI 번역
 │ 또는 개별 상품 클릭
 ▼
ED-02 편집 상세
 │ 기본 정보 탭: 상품명/설명/옵션 AI 번역
 │ 가격 탭: 마진율 조정
 │ 이미지 탭: 썸네일/상세 AI 번역
 │ → "등록하기" 클릭
 ▼
RG-01 판매 중인 상품
 │ 등록 프로그레스 확인 → 완료
 │ 상품 클릭
 ▼
RG-02 상품 상세
 │ 판매 정보 확인
 │ 모니터링 ON
 │ 완료!
```

### 핵심 분기점

| 지점 | 조건 | 경로 A | 경로 B |
|------|------|--------|--------|
| S-01 | 기존 사용자 | → SC-01 (온보딩 스킵) | → S-02 (온보딩 시작) |
| S-05 | 설치 여부 | "설치하기" → 대기 → 완료 | "나중에" → SC-01 직행 |
| SC-01 | 수집 방식 | 자동 수집 → SC-03 | 직접 수집 → SC-02 |
| ED-01 | 등록 방식 | 개별 → ED-02 → 등록 | 일괄 → RG-01 직행 |
| ED-02 | 번역 상태 | 완료 → "등록하기" 활성 | 미완료 → "등록하기" 비활성 |

## 플로우 2: URL로 단건 수집 → 편집 → 등록

가장 간단한 수집 경로. 특정 상품 URL을 알고 있을 때.

```
SC-01 수집 홈
 │ "직접 수집" 탭
 ▼
SC-02 URL 수집 (또는 SC-01 인라인)
 │ URL 1개 입력
 │ 실시간 검증: ✅ 올리브영
 │ "총 1건 수집 시작하기"
 │ 수집 진행 (1.5~3초)
 │ ✅ 완료
 ▼
ED-01 편집 목록
 │ 새 상품 표시 (파란 점 = 신규)
 │ 상품 클릭
 ▼
ED-02 편집 상세
 │ AI 번역 실행
 │ 가격 확인/조정
 │ "등록하기"
 ▼
RG-01 등록 결과
```

## 플로우 3: 자동 수집 설정 → 알림 → 편집 → 등록

반복적인 일상 워크플로우. 한 번 설정하면 매일 자동 수집.

```
SC-01 수집 홈
 │ "자동 수집" 탭 → "추가하기"
 ▼
SC-03 자동 수집 설정
 │ Step 1: 올리브영 선택
 │ Step 2: 뷰티 - 스킨케어 / 인기상품
 │ Step 3: 매일 07:00
 │ "저장"
 ▼
SC-01 수집 홈
 │ 스케줄 카드 표시됨
 │ (다음 날 07:00에 자동 실행)
 │
 │ ── 다음 날 아침 ──
 │
 │ 알림 패널 (G-02): "올리브영 수집 완료 (50건)"
 │ 사이드바 뱃지: 수집된 상품 (50)
 │ 알림 클릭
 ▼
ED-01 편집 목록 (?focusJobId=...)
 │ 해당 수집건 자동 선택
 │ "AI 번역" 일괄 실행
 │ 번역 완료 알림 확인
 │ 편집 완료 상품 선택 → "등록하기"
 ▼
RG-01 등록 결과
```

## 플로우 4: 등록 실패 → 원인 확인 → 수정 → 재등록

등록 실패 시 대응 경로.

```
RG-01 등록 결과
 │ 등록 프로그레스: 10건 중 2건 실패
 │ "문제 발생" 탭에서 실패 상품 확인
 │ 실패 상품 클릭
 ▼
RG-02 상품 상세
 │ 에러 정보 확인:
 │ - api_error: API 연동 오류
 │ - missing_required: 필수 항목 누락
 │ - category_mismatch: 카테고리 불일치
 │ - image_spec: 이미지 규격 미달
 │ - policy_violation: 정책 위반
 │ - server_timeout: 서버 타임아웃
 │
 │ "편집" 클릭
 ▼
RG-03 등록 편집
 │ 문제 항목 수정
 │ "저장하기"
 ▼
RG-02 상품 상세
 │ (수정 반영 확인)
 │ → ED-01에서 재등록 가능
```

## 플로우 5: 판매 중 상품 가격 변동 → 알림 → 대응

모니터링 활성 상품에서 이슈 발생 시.

```
RG-01 등록 결과
 │ 상품 등록 후 모니터링 ON
 │
 │ ── 며칠 후 ──
 │
 │ 소싱처 원가 상승 감지
 │
 ▼
┌─────────────────────────┬───────────────────────────┐
│ 자동 조정 가능            │ 역마진 발생                │
│                         │                           │
│ 알림 (G-02):             │ 알림 (G-02):               │
│ "판매가가 자동으로         │ "역마진이 발생했어요.       │
│  ¥N로 조정됐어요"         │  판매가 조정이 필요해요"    │
│                         │                           │
│ → 확인만 하면 OK          │ → 알림 클릭               │
│                         │ → RG-02 상품 상세          │
│                         │ → "편집" → RG-03           │
│                         │ → 판매가 조정 → 저장       │
└─────────────────────────┴───────────────────────────┘
```

### 품절 감지 시

```
소싱처 품절 감지
 ▼
┌─────────────────────────┬───────────────────────────┐
│ 자동 중지 설정됨          │ 수동 대응 필요              │
│                         │                           │
│ 알림 (G-02):             │ 알림 (G-02):               │
│ "품절이 감지돼 자동으로    │ "품절이 감지됐어요.         │
│  판매가 중지됐어요"       │  판매 일시중지가 필요해요"  │
│                         │                           │
│ → 확인만 하면 OK          │ → 알림 클릭               │
│ → 재입고 시 자동 재개     │ → RG-02 상품 상세          │
│                         │ → "판매 중지" 클릭          │
└─────────────────────────┴───────────────────────────┘
```

## 플로우 6: 크롬 익스텐션으로 수집

쇼핑몰을 브라우징하면서 원하는 상품을 직접 수집.

```
(쇼핑몰 브라우징 — 크롬 익스텐션 활성)
 │ 상품 페이지에서 DayZero 아이콘 클릭
 │ → 수집 완료 알림
 │
 ▼
ED-01 편집 목록
 │ 새 상품 표시
 │ (이후 플로우 2와 동일)
```

> 주의: 크롬 익스텐션 ↔ 웹앱 통신은 프로토타입에서 시뮬레이션만 구현.
> 실제 익스텐션 개발은 프로덕션 범위.

## 플로우 7: 설정 변경 → 기존 상품 가격 재계산

마진율이나 배송비를 변경하면 기존 상품 판매가에 영향.

```
ST-01 설정 (판매 설정 탭)
 │ 기본 마진율 25% → 30% 변경 (또는 환율, 배송비 등 가격 영향 필드 변경)
 │ "저장하기" 클릭
 ▼
ApplyScopeModal (적용 범위 선택):
 │ ┌─────────────────────────────────────────────┐
 │ │ "어떤 항목을 적용하시겠어요?"                  │
 │ │ ☑ 마진율  ☑ 환율  ☑ 배송비 (변경된 항목 표시) │
 │ │                                             │
 │ │ "어떤 상품에 적용할까요?"                      │
 │ │ ● 새로 수집하는 상품부터 적용 (기본값)         │
 │ │ ○ 모든 상품에 적용 (수집+등록 상품 일괄 변경)  │
 │ │                                             │
 │ │ ⚠️ "모든 상품" 선택 시 경고:                  │
 │ │ "수집한 상품과 등록한 상품의 판매가가           │
 │ │  일괄 변경됩니다."                            │
 │ │                                             │
 │ │        [취소]  [저장하기]                     │
 │ └─────────────────────────────────────────────┘
 │
 ├── "새로 수집하는 상품부터 적용" 선택 시:
 │    → 설정값만 저장, 기존 상품 가격 변동 없음
 │    → 이후 새로 수집되는 상품부터 변경된 설정 적용
 │
 └── "모든 상품에 적용" 선택 시:
      → ED-01 편집 목록의 모든 상품 판매가 재계산
      → RG-01 등록 상품의 판매가도 재계산
      → 토스트: "{변경 항목}이(가) 변경되어 N건의 판매가가 재계산됩니다."
 ▼
토스트: "설정이 저장되었습니다"
```

## 전체 화면 연결 다이어그램

```
[S-01] ─→ [S-02] ─→ [S-03] ─→ [S-05]
                                                │
                                                ▼
[ST-01] ←─── 사이드바 ───→ [SC-01] ←─────────────┘
                              │
                    ┌─────────┼─────────┐
                    ▼         ▼         ▼
                 [SC-02]   [SC-03]   (익스텐션)
                    │         │         │
                    └─────────┼─────────┘
                              ▼
                           [ED-01]
                              │
                              ▼
                           [ED-02]
                              │
                              ▼
                           [RG-01]
                              │
                    ┌─────────┼─────────┐
                    ▼                   ▼
                 [RG-02]             [RG-03]
                    │                   │
                    └───────────────────┘
```

### GNB(사이드바)에서 직접 이동 가능한 경로
- 수집하기 → SC-01
- 수집된 상품 → ED-01
- 판매 중인 상품 → RG-01
- 설정 → ST-01 (하위 탭 3개)

### 알림 패널(G-02)에서 이동 가능한 경로
- 수집 완료 → ED-01
- 번역 완료 → ED-01
- 등록 완료/실패 → RG-01
- 가격 변동/역마진/품절/재입고 → RG-02 (해당 상품)
