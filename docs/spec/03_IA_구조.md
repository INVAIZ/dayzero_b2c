## 네비게이션 구조

### 사이드바 (G-01)

온보딩 완료 후 모든 메인 화면에 좌측 고정으로 표시됩니다.

```
┌─────────────────────┐
│ [DayZero 로고]       │  ← 클릭 시 /sourcing
│                     │
│ 🔍 수집하기          │  ← /sourcing (활성: 배경 하이라이트)
│ 📋 수집된 상품 (N)   │  ← /editing (N = 미처리 상품 수, 뱃지)
│ 📦 판매 중인 상품     │  ← /registration
│                     │
│ ⚙️ 설정              │  ← 클릭 시 하위 메뉴 펼침
│   ├ 판매 설정        │  ← /settings/sales
│   ├ Qoo10 연동       │  ← /settings/qoo10
│   └ 계정             │  ← /settings/account
│                     │
└─────────────────────┘
```

**활성 상태 판정:**
- `수집하기`: pathname이 `/sourcing`으로 시작
- `수집된 상품`: pathname이 `/editing`으로 시작
- `판매 중인 상품`: pathname이 `/registration`으로 시작
- `설정`: pathname이 `/settings`으로 시작 → 하위 메뉴 자동 펼침

**뱃지:**
- `수집된 상품` 옆에 미처리 상품 수 표시 (unprocessedProductCount)
- 수가 증가하면 뱃지 애니메이션 (0.6초)

> **알림 진입점:** 사이드바에는 알림 아이콘이 없음. 알림은 우측 하단 FAB(Bell 아이콘)에서만 접근 (→ G-02 참조).

### 레이아웃 구성

```
┌──────────┬─────────────────────────────────┐
│          │                                 │
│ Sidebar  │         Main Content            │
│ (280px)  │    (padding: 48px 64px)         │
│ 고정     │    (max-width: 1200px)          │
│          │                                 │
│          │                                 │
└──────────┴─────────────────────────────────┘
```

- **사이드바**: 280px 고정, 좌측 fixed
- **메인 영역**: margin-left 280px, 배경 흰색
- **콘텐츠**: max-width 1200px, 가로 중앙 정렬
- **알림 패널**: 우측에서 슬라이드 인 (오버레이)
- **토스트**: 하단 중앙 고정

### 온보딩 흐름 (사이드바 없음)

온보딩 화면(S-01~S-05)은 사이드바/MainLayout 없이 독립 전체 화면으로 표시됩니다.

```
S-01 로그인
  ↓ 로그인 성공 (신규)
S-02 Qoo10 JP API 연동
  ↓ 연동 완료
S-03 배송지/기본 정보
  ↓ 정보 입력 완료
S-05 크롬 익스텐션 설치
  ↓ 설치 완료 또는 건너뛰기
SC-01 수집 홈 (메인 진입)
```
