# 데이터 정의

> 각 화면에서 사용하는 핵심 데이터 엔티티, 상태 전이 규칙, 화면별 데이터 매핑을 정의합니다.
> 기술 구현(DB 스키마, API)이 아닌 **기획 관점의 데이터 구조**입니다.

---

## 1. 핵심 엔티티

### 상품 (ProductDetail)

수집부터 등록까지 전체 파이프라인에서 사용되는 중심 엔티티.

| 필드 | 설명 | 예시 |
|------|------|------|
| **원문 (한국어)** | | |
| titleKo | 한국어 상품명 | "[닥터지] 레드 블레미쉬 클리어 수딩 크림 70ml" |
| descriptionKo | 한국어 상세 설명 | HTML 또는 텍스트 |
| options | 옵션 목록 (nameKo, nameJa, stock, priceKrw, isRepresentative) | [{nameKo: "70ml", nameJa: "70ml", stock: 50, priceKrw: 18900, isRepresentative: true}] |
| **번역본 (일본어)** | | |
| titleJa | 일본어 상품명 | "ドクタージー レッドブレミッシュ..." |
| descriptionJa | 일본어 상세 설명 | |
| **이미지** | | |
| thumbnails | 썸네일 이미지 (최대 10장) | [{url, translatedUrl, translationStatus, backgroundRemoved}] |
| detailImages | 상세 이미지 (최대 30장) | 동일 구조 |
| thumbnailTranslated | 썸네일 번역 완료 여부 | true/false |
| detailPageTranslated | 상세페이지 번역 완료 여부 | true/false |
| **가격** | | |
| originalPriceKrw | 소싱처 원가 (₩) | 18900 |
| salePriceJpy | Qoo10 판매가 (¥) | 2980 |
| **카테고리** | | |
| qoo10CategoryId | Qoo10 카테고리 코드 | "100000012" |
| qoo10CategoryPath | 표시용 카테고리 경로 | "스킨케어 > 기초화장품 > 에센스" |
| aiRecommendedCategoryId | AI 최초 추천 코드 | 변경 추적용 |
| **브랜드** | | |
| brand | 브랜드명 | "닥터지" |
| brandMatchStatus | Qoo10 매칭 상태 | 'matched' / 'unmatched' / 'none' |
| **소싱 정보** | | |
| provider | 소싱처 | '올리브영' / '쿠팡' / '다이소' 등 12개 |
| sourceUrl | 소싱처 원본 URL | "https://www.oliveyoung.co.kr/..." |
| thumbnailUrl | 대표 썸네일 URL | |
| **물류** | | |
| weightKg | 무게 (kg) | 0.3 |
| weightSource | 무게 출처 | 'crawled' / 'ai' / 'manual' |
| priceSource | 가격 출처 | 'crawled' / 'ai' / 'manual' |
| shippingType | 발송 유형 | 'standard' / 'sameday' / 'preorder' |
| shippingDays | 발송까지 영업일 | 3 |
| **상태** | | |
| translationStatus | 번역 상태 | 'pending' / 'processing' / 'completed' / 'failed' |
| editStatus | 편집 상태 | 'pending' / 'processing' / 'completed' / 'failed' |
| isRead | 사용자가 확인했는지 | true/false |
| createdAt | 수집일 (ISO) | "2026-04-07T09:00:00Z" |
| lastSavedAt | 마지막 저장일 | |

### 수집 작업 (SourcingJob)

URL 수집 또는 자동 수집 실행 단위.

| 필드 | 설명 |
|------|------|
| id | 작업 고유 ID |
| type | 'URL' (직접) / 'AUTO' (자동) |
| provider | 소싱처 |
| categorySummary | 카테고리 요약 (예: "뷰티 인기상품") |
| status | 'queued' / 'running' / 'completed' / 'failed' |
| totalCount | 전체 수집 대상 수 |
| currentCount | 현재까지 수집된 수 |
| createdAt | 시작 시간 |
| completedAt | 완료 시간 |
| duplicateCount | 중복 건수 |

### 자동 수집 스케줄 (AutoSchedule)

반복 수집을 위한 스케줄 설정.

| 필드 | 설명 | 예시 |
|------|------|------|
| id | 스케줄 고유 ID | |
| provider | 소싱처 | '올리브영' |
| categoryPath | 카테고리 | "뷰티 - 스킨케어" |
| criteria | 수집 기준 | '인기상품' / '전체상품' |
| targetCount | 수집 목표 건수 | 50 |
| timeString | 수집 시간 | "07:00" |
| isActive | 활성화 여부 | true/false |
| lastRunAt | 마지막 실행 시간 | |

### 번역 작업 (TranslationJob)

AI 번역 실행 단위.

| 필드 | 설명 |
|------|------|
| id | 작업 고유 ID |
| productId | 대상 상품 ID |
| productTitleKo | 상품명 (표시용) |
| status | 'queued' / 'processing' / 'completed' / 'failed' |
| targets | 번역 대상 ['title', 'description', 'options', 'thumbnail', 'detailPage'] |
| isRead | 완료 알림 읽음 여부 |

### 등록 결과 (RegistrationResult)

Qoo10 등록 후 상품별 결과.

| 필드 | 설명 |
|------|------|
| id | 결과 고유 ID |
| productId | 원본 상품 ID |
| product | 등록 시점 상품 스냅샷 (ProductDetail) |
| status | 'success' / 'failed' |
| marketplace | 'qoo10_jp' |
| registeredAt | 등록 시간 |
| qoo10ItemCode | Qoo10 상품 코드 (성공 시) |
| qoo10ProductUrl | Qoo10 상품 URL (성공 시) |
| error | 에러 정보 (실패 시) |
| salesStatus | 판매 상태: 'active' / 'paused' |
| pauseReason | 중지 사유: 'auto' (품절) / 'manual' |
| monitoring | 모니터링 정보 (MonitoringInfo) |

### 등록 에러 (RegistrationError)

| 에러 타입 | 설명 |
|-----------|------|
| api_error | Qoo10 API 연동 오류 |
| missing_required | 필수 항목 누락 |
| category_mismatch | 카테고리 불일치 |
| image_spec | 이미지 규격 미달 |
| policy_violation | 정책 위반 |
| server_timeout | 서버 타임아웃 |

### 모니터링 정보 (MonitoringInfo)

등록 상품의 가격/재고 변동 추적.

| 필드 | 설명 |
|------|------|
| status | 'active' / 'inactive' |
| lastCheckResult | 'normal' / 'price_changed' / 'negative_margin' / 'out_of_stock' / 'restocked' |
| lastCheckAt | 마지막 확인 시간 |
| nextCheckAt | 다음 확인 예정 |
| currentSourcePriceKrw | 현재 소싱처 가격 |
| priceHistory | 최근 30일 가격 이력 [{date, sourcePriceKrw, stockStatus, marginPercent}] |
| issueDescription | 문제 상세 설명 |
| activityLog | 활동 타임라인 [{date, type, description, details}] |

### 모니터링 활동 로그 이벤트 유형

| type | 설명 | 아이콘 색상 |
|------|------|------------|
| monitoring_started | 자동확인 시작 | 파랑 |
| price_changed | 원가 변동 + 판매가 자동 조정 | 주황 |
| out_of_stock | 품절 → 자동 일시중지 | 빨강 |
| restocked | 재입고 → 자동 재개 | 초록 |
| negative_margin | 역마진 발생 | 빨강 |
| error | 소싱처 접근 불가 등 | 회색 |

## 2. 상품 생명주기

```
수집됨 (pending)
  │
  │ AI 번역 실행
  ▼
번역 중 (processing)
  │
  ├── 성공 ──→ 번역 완료 (completed)
  │                │
  │                │ 사용자 편집
  │                ▼
  │            편집 완료 (editStatus: completed)
  │                │
  │                │ "등록하기"
  │                ▼
  │            등록 중 (processing)
  │                │
  │           ┌────┴────┐
  │           ▼         ▼
  │       등록 성공   등록 실패
  │           │         │
  │           ▼         │ 수정 후 재등록
  │       판매 중       └──→ 편집 완료로 복귀
  │           │
  │     모니터링 ON
  │           │
  │    ┌──────┼──────┬──────────┐
  │    ▼      ▼      ▼          ▼
  │  정상  가격변동  역마진     품절
  │    │      │      │          │
  │    │   자동조정  수동대응  자동중지
  │    │      │      │          │
  │    └──────┴──────┘     재입고→재개
  │
  └── 실패 ──→ 번역 실패 (failed)
                  │ 재시도 가능
                  └──→ 번역 중
```

### 번역 완료 판정 기준
- `isTextTranslated(product)`: titleJa + descriptionJa 모두 존재
- `isFullyTranslated(product)`: titleJa + descriptionJa + 모든 옵션 nameJa 존재
- ED-01의 "편집 완료" 탭: `isFullyTranslated` 기준

### 등록 가능 조건
- `isTextTranslated(product)` === true (상품명 + 설명 번역 완료)
- ED-02 헤더 "등록하기" 버튼 활성화 조건

## 6. 가격 계산 공식

```
원가 (₩) = originalPriceKrw
작업비 (₩) = prepCost (온보딩 설정)
국내배송비 (₩) = domesticShipping
해외배송비 (¥) = 배송대행사 무게별 테이블 lookup
해외배송비 (₩) = 해외배송비(¥) × 환율

비용 합계 (₩) = 원가 + 작업비 + 국내배송비 + 해외배송비(₩)
마진 (₩) = 비용 합계 × (마진율 / 100)
수수료 보전 전 (₩) = 비용 합계 + 마진

판매가 (¥) = round(수수료 보전 전(₩) / 환율 / (1 - 수수료율))
Qoo10 수수료 = 판매가(¥) × 12%
수령액 (¥) = 판매가 - 수수료
수령액 (₩) = 수령액(¥) × 환율
순수익 (₩) = 수령액(₩) - 비용 합계
```

**상수:**
- Qoo10 수수료율: 12% (`PLATFORM_FEE_RATE`)
- 환율: `EXCHANGE_RATE` (프로토타입 고정값)
