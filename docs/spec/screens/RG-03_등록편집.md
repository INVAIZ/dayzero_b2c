# [RG-03] 등록 상품 편집

## 화면 목적
이미 Qoo10에 등록된 상품의 정보를 수정한다.
ED-02와 동일한 편집 UI를 재사용하되, 저장 시 등록 상품에 반영하는 별도 로직을 가진다.

## 진입 경로
- RG-02 "수정하기" 버튼 클릭
- URL 직접 접근 (`/registration/:resultId/edit`)
- URL `?tab=` 파라미터로 초기 탭 지정 가능 (예: `?tab=price`)

## 레이아웃
Sidebar 직접 포함 (MainLayout 미사용, ED-02와 동일 구조)

```
┌──────────┬──────────────────────────────────────────┐
│          │ [← 돌아가기] | [썸네일+소싱처+상품명]      │
│ Sidebar  │                        [저장하기] 버튼    │
│          ├──────────────────────────────────────────┤
│          │ [기본 정보] [가격 정보] [썸네일·상세페이지] │
│          │                                          │
│          │ ── ED-02와 동일한 탭 콘텐츠 ──             │
│          │                                          │
└──────────┴──────────────────────────────────────────┘
```

## 핵심 차이점 (ED-02 대비)

| 항목 | ED-02 | RG-03 |
|------|-------|-------|
| 데이터 소스 | editingStore.products | registrationStore → 임시로 editingStore에 복사 |
| 저장 | editingStore 직접 반영 | registrationStore의 등록 상품에 반영 (`updateRegisteredProduct`) |
| 헤더 버튼 | "등록하기" | "저장하기" |
| 뒤로가기 | /editing | /registration/:resultId (RG-02) |
| 변경 감지 | 없음 | isDirty 추적 → 미저장 이탈 시 경고 |
| 원가 | 수집 시 원가 | 모니터링 현재 원가 반영 (`monitoring?.currentSourcePriceKrw`) |
| 임시 ID | 원본 product.id | `__reg_edit__{resultId}` |
| 언마운트 | — | editingStore에서 임시 상품 자동 제거 |

## 섹션 1: 스티키 헤더

### 구조
- `position: sticky`, `top: 0`, `zIndex: 10`
- 배경: `bg.surface`, 하단 테두리: `border.default`, 그림자: `shadow.sm`
- 최대 너비: 1200px, 가운데 정렬

### 좌측: 돌아가기 버튼
- ArrowLeft (14px) + `돌아가기`
- 색상: `text.tertiary`, 호버: `text.primary`
- 클릭 동작: isDirty 여부에 따라 분기 (아래 참조)

### 중앙: 상품 정보
- 구분선 (1px 세로, 32px 높이)
- 썸네일 (40x40, `radius.md`)
- 소싱처 로고 (18x18, `radius.xs`)
- 상품명 — `product.titleJa` 있으면 일본어, 없으면 한국어 (`stripPrefix` 적용). 볼드, 말줄임

### 우측: 판매 상태 + 저장하기 버튼

#### 판매 일시중지/재개 버튼

| salesStatus | 텍스트 | 아이콘 | 스타일 |
|-------------|--------|--------|--------|
| `active` | `판매 일시중지` | PauseCircle (14px) | `colors.text.tertiary`, `border: 1px solid colors.border.default`, 호버: `colors.text.primary` |
| `paused` | `판매 재개` | PlayCircle (14px) | `colors.primary`, `border: 1px solid colors.primaryBorder`, 호버: `colors.primaryDark` 텍스트 |

- 클릭 시 RG-02와 동일한 일시중지/재개 확인 모달 표시
- 확인 후 `salesStatus` 변경 → registrationStore에 즉시 반영 (isDirty와 무관)

#### 저장하기 버튼

| 상태 | 배경 | 텍스트 색상 | 커서 | opacity |
|------|------|------------|------|---------|
| `isDirty === true` | `colors.primary` | `bg.surface` (흰색) | `pointer` | 1 |
| `isDirty === false` | `colors.border.default` | `text.muted` | `not-allowed` | 0.6 |

- 호버 (isDirty일 때): `primaryHover`
- 전이: `background 0.2s`
- 클릭: isDirty일 때만 저장 확인 모달 열기

## 섹션 2: 탭 + 콘텐츠

### 탭 바 (EditingTabBar)
- ED-02와 동일한 컴포넌트 재사용
- 3개 탭: `basic` (기본 정보), `price` (가격 정보), `images` (썸네일·상세페이지)
- URL `?tab=` 파라미터로 초기 탭 결정 (`initialTab`)

### 탭별 콘텐츠

| 탭 | 컴포넌트 | 비고 |
|----|---------|------|
| `basic` | `<BasicEditTab product={product} hideProgress />` | `hideProgress` prop으로 등록 프로그레스 숨김 |
| `price` | `<PriceEditTab product={product} autoSave={false} onChanged={...} />` | `autoSave=false` (자동 저장 비활성), `onChanged` 콜백으로 isDirty 설정 |
| `images` | `<ThumbnailEditTab>` + 구분선 + `<DetailImageEditTab>` | 세로 배치, `gap: spacing[12]`, 구분선: `border.default` |

## 마운트/언마운트 라이프사이클

### 마운트 (`useEffect`, deps: `[resultId]`)

1. `registrationStore.jobs`에서 `resultId`에 해당하는 `result` 조회
2. result 없으면 조기 반환
3. editingStore에 `tempProductId` (`__reg_edit__{resultId}`)로 이미 존재하는지 확인
4. 없으면 `addProduct` 호출:
   - `result.product` 복사 + `id: tempProductId`
   - `monitoring?.currentSourcePriceKrw`가 있으면 `originalPriceKrw`를 이 값으로 덮어씀 (모니터링 현재 원가 반영)
5. `setCurrentEditProduct(tempProductId)` — 편집 대상 상품 설정

### 언마운트 (cleanup 함수)

1. `setCurrentEditProduct(null)` — 편집 대상 해제
2. `removeProduct(tempProductId)` — editingStore에서 임시 상품 제거

## 변경 감지 (isDirty)

### 감지 대상 필드
```
titleKo, titleJa, descriptionKo, descriptionJa,
salePriceJpy, originalPriceKrw, qoo10CategoryId, weightKg,
options (nameKo, nameJa, priceKrw, stock 변경 포함)
```

> 옵션 데이터 변경(이름 수정, 가격 변경, 추가/삭제)도 isDirty 감지 대상입니다.
> 옵션은 `JSON.stringify(product.options)`로 스냅샷 비교합니다.

### 동작 원리
1. 마운트 시 위 필드들을 `JSON.stringify`하여 `snapshotRef`에 초기 스냅샷 저장
2. `product`가 변경될 때마다 현재 스냅샷과 초기 스냅샷 비교
3. 달라지면 `isDirty = true`, `isDirtyRef.current = true`
4. 저장 성공 시 `markClean()` 호출 → `isDirty = false`, 스냅샷 재설정

### isDirtyRef 사용 이유
- `useRef`로 최신 dirty 상태 유지 — 클로저 문제 방지
- `handleBack` 등에서 즉시 최신 상태 참조 가능

## 미저장 이탈 방지

### "← 돌아가기" 클릭 시

| isDirty | 동작 |
|---------|------|
| `false` | 즉시 `/registration/{resultId}` (RG-02)로 이동 |
| `true` | 이탈 확인 모달 열기 (`isLeaveModalOpen = true`) |

### 이탈 확인 모달에서

| 버튼 | 동작 |
|------|------|
| `나가기` (confirmText) | 모달 닫기 → `isDirtyRef.current = false` → `pendingLeave()` 실행 (RG-02로 이동) |
| `계속 수정하기` (cancelText) | 모달 닫기 + `pendingLeave` 초기화 → 현재 페이지 유지 |

## 저장 플로우

### 1단계: 저장 버튼 클릭
- `isDirty === true`일 때만 동작
- `isSaveModalOpen = true`

### 2단계: 저장 확인 모달

| 속성 | 값 |
|------|---|
| 제목 | `이대로 Qoo10에 반영할까요?` |
| 설명 | `수정한 내용이 Qoo10 JP 상품 페이지에 즉시 반영됩니다.` |
| type | `info` |
| 확인 버튼 | `반영하기` |
| 취소 버튼 | `취소` |

### 3단계: handleSave 실행
1. 임시 상품(`product`)을 복사하되 `id`를 원본 `result.product.id`로 복원
2. `updateRegisteredProduct(resultId, savedProduct)` 호출 → registrationStore에 반영
3. `markClean()` — isDirty 초기화, 스냅샷 재설정
4. 모달 닫기
5. 토스트: `"저장되었어요"` (5초 후 자동 소멸)

## 이동 경로

| 액션 | 목적지 | 조건 |
|------|--------|------|
| `← 돌아가기` (isDirty=false) | `/registration/:resultId` (RG-02) | 바로 이동 |
| `← 돌아가기` (isDirty=true) → "나가기" | `/registration/:resultId` (RG-02) | 이탈 모달 확인 후 |
| `← 돌아가기` (isDirty=true) → "계속 수정하기" | 현재 페이지 유지 | — |
| 저장 확인 → "반영하기" | 현재 페이지 유지 (저장 후 계속 편집) | isDirty 초기화됨 |
| 상품 없음 → "목록으로 돌아가기" | `/registration` (RG-01) | — |
