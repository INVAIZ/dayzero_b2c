# [SC-01] 수집 홈

## 화면 목적
상품 수집의 메인 허브. 자동 수집/직접 수집 탭으로 수집 방식을 선택하고,
등록된 자동 수집 스케줄을 관리한다.

## 진입 경로
- 온보딩 완료 후 최초 진입
- 사이드바 "수집하기" 클릭
- SC-02, SC-03에서 뒤로가기

## 레이아웃
MainLayout (사이드바 + 메인 영역). 하단 패딩 100px.
페이지 진입 시 `fadeInUp 0.6s ease` 애니메이션 적용.

```
┌──────────────────────────────────────┐
│ 상품 수집                             │
│ 수집 방식을 선택하고 인기 상품을...      │
│                                      │
│ [자동 수집] [직접 수집]  ← 세그먼트 탭  │
│                                      │
│ ── 자동 수집 탭 ──                    │
│ 안내 배너 (조건부)                     │
│                                      │
│ 등록된 자동 수집 목록     [+ 추가]     │
│ [필터: 전체 | 올리브영 | 쿠팡 | ...]    │
│                                      │
│ ┌─ 스케줄 카드 (드래그 정렬) ────────┐│
│ │ [로고] 올리브영에서 뷰티 인기상품... ││
│ │ 매일 07:00에 수집해요.   [🗑][토글] ││
│ └────────────────────────────────────┘│
│                                      │
│ ── 직접 수집 탭 ──                    │
│ (UrlSourcingContent 인라인 렌더링)     │
└──────────────────────────────────────┘
```

## 컴포넌트 구조

- `SourcingPage.tsx` — 메인 페이지 컴포넌트
- `UrlSourcingContent.tsx` — 직접 수집 탭에서 인라인 렌더링 (SC-02와 동일)

상태는 `useSourcingStore`에서 관리:
- `schedules: AutoSchedule[]` — 등록된 자동 수집 스케줄 목록
- `selectedAutoFilter: string` — 현재 선택된 소싱처 필터 (기본값 `'전체'`)
- `toggleSchedule(id)` — 스케줄 활성/비활성 전환
- `deleteSchedule(id)` — 스케줄 삭제
- `reorderSchedules(schedules)` — 드래그 정렬 후 순서 반영

로컬 상태:
- `activeTab: 'auto' | 'url'` — 현재 활성 탭 (기본값 `'auto'`)
- `draggedId: string | null` — 드래그 중인 스케줄 ID
- `hoveredId: string | null` — 호버 중인 스케줄 ID
- `deleteModal: { isOpen: boolean; scheduleId: string | null }` — 삭제 확인 모달 상태
- `limitModal: boolean` — 한도 초과 모달 상태
- `visibleCount: number` — 현재 표시되는 스케줄 수 (기본값 10)

### 5. 더 보기 버튼

표시 조건: `filteredSchedules.length > visibleCount` (`hasMore === true`)

계산 로직:
- `filteredSchedules` = 선택된 필터가 "전체"이면 전체 schedules, 아니면 해당 provider만 필터링
- `clampedVisible` = `Math.min(visibleCount, filteredSchedules.length)`
- `visibleSchedules` = `filteredSchedules.slice(0, clampedVisible)`

| 속성 | 값 |
|------|-----|
| 텍스트 | `더 보기 ({filteredSchedules.length - visibleCount}건 더)` |
| width | `100%` |
| padding | `spacing['3']` (상하), 0 (좌우) |
| background | 기본: `none`, 호버: `colors.bg.subtle` |
| border | `1px solid colors.border.default` |
| borderRadius | `radius.lg` |
| fontSize | `font.size.base` |
| fontWeight | `font.weight.medium` |
| color | 기본: `colors.text.tertiary`, 호버: `colors.text.secondary` |
| transition | `background 0.15s, color 0.15s` |

클릭 시 `setVisibleCount(v => v + 10)` — 10건씩 추가 표시.

## 동작 정의

### 탭 전환

- **Given**: 수집 홈에 있음
- **When**: "자동 수집" / "직접 수집" 탭 클릭
- **Then**: `setActiveTab('auto' | 'url')` → 해당 탭 콘텐츠 표시

### 자동 수집 추가

- **Given**: 자동 수집 탭
- **When**: `+ 자동 수집 추가` 또는 빈 상태의 `추가하기` 클릭
- **Then**:
  - `schedules.length < 10`: `/sourcing/auto` (SC-03) 이동
  - `schedules.length >= 10`: 한도 모달 표시

### 스케줄 토글

- **Given**: 스케줄 카드
- **When**: 토글 스위치 클릭
- **Then**: `toggleSchedule(schedule.id)` → `isActive` 반전. 트랙 배경/노브 위치 `0.2s` 전환 애니메이션.

### 스케줄 삭제

- **Given**: 스케줄 카드 호버 상태
- **When**: Trash2 아이콘 클릭
- **Then**: 삭제 확인 모달 표시 → 확인 시 `deleteSchedule(scheduleId)` 실행

### 스케줄 편집

- **Given**: 스케줄 카드
- **When**: 카드 영역 클릭 (토글/삭제 외 영역)
- **Then**: `/sourcing/auto?edit={schedule.id}` (SC-03 편집 모드) 이동

### 스케줄 드래그 정렬

- **Given**: 스케줄 목록 (2건 이상)
- **When**: GripVertical 핸들을 드래그
- **Then**:
  1. `onDragStart`: `setDraggedId(schedule.id)` → 해당 카드 `opacity: 0.5`, `border-color: colors.primary`
  2. `onDragOver`: `e.preventDefault()` (드롭 허용)
  3. `onDrop`: `handleDrop(targetId)` — 배열에서 fromIdx → toIdx 이동, `reorderSchedules()` 호출
  4. `onDragEnd`: `setDraggedId(null)` — opacity/border 복원

### 소싱처 필터

- **Given**: 자동 수집 탭, 스케줄 있음
- **When**: 필터 칩 클릭
- **Then**:
  1. `setSelectedFilter(filter)`
  2. `setVisibleCount(10)` — 더 보기 상태 초기화
  3. 선택된 필터에 맞는 스케줄만 표시 (체크 아이콘 + 텍스트 토글 `0.3s ease` 애니메이션)

### 더 보기

- **Given**: `filteredSchedules.length > visibleCount`
- **When**: `더 보기` 버튼 클릭
- **Then**: `setVisibleCount(v => v + 10)` → 10건씩 추가 표시

## 애니메이션

| 대상 | 애니메이션 | 타이밍 |
|------|-----------|--------|
| 페이지 헤더 | `fadeInUp 0.6s ease` | 마운트 시 |
| 자동 수집 탭 콘텐츠 | `fadeInUp 0.4s ease` | 탭 전환 시 (key="auto") |
| 스케줄 목록 | `fadeInUp 0.3s ease` | 필터 변경 시 (key=selectedFilter) |
| 필터 빈 결과 | `fadeIn 0.2s ease` | 필터 결과 0건 시 |
| 필터 칩 전환 | `all 0.3s ease` (체크 아이콘, 텍스트, 배경/테두리) | 필터 클릭 시 |
| 드래그 중 카드 | `opacity 0.15s, border-color 0.15s` | 드래그 시작/종료 |
| 드래그 핸들/삭제 버튼 | `opacity 0.15s` | 카드 호버 진입/이탈 |
| 토글 스위치 | `background 0.2s` (트랙), `left 0.2s` (노브) | 토글 클릭 시 |
| 삭제 버튼 호버 | `color 0.15s` (muted → danger) | 삭제 아이콘 호버 |
| 빈 상태 CTA 호버 | `transform 0.2s` (scale 1 → 1.02) | CTA 버튼 호버 |
| 더 보기 호버 | `background 0.15s, color 0.15s` | 버튼 호버 |

인라인 키프레임 정의 (`<style>` 태그):
```css
@keyframes fadeInUp { from { opacity: 0; transform: translateY(8px); } to { opacity: 1; transform: translateY(0); } }
@keyframes fadeIn { from { opacity: 0; } to { opacity: 1; } }
```
