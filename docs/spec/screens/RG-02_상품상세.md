# [RG-02] 등록 상품 상세

## 화면 목적
Qoo10에 등록된 개별 상품의 상세 정보, 판매 상태, 모니터링 결과를 확인한다.
가격 변동 이력, 활동 로그를 통해 상품 건강 상태를 파악한다.

## 진입 경로
- RG-01 상품 행 클릭 (`/registration/:resultId`)
- 알림 패널에서 가격 변동/품절/재입고 알림 클릭

**헤더 우측**: `수정하기` 버튼 (PenLine 11px + 텍스트, 클릭 → `/registration/{resultId}/edit` RG-03)

| 행 (InfoRow) | label | value | 비고 |
|-------------|-------|-------|------|
| 판매가 | `판매가` | `¥{product.salePriceJpy.toLocaleString()}` | highlight |

## 섹션 6: 가격 변동 이력 차트 (PriceHistorySection)

### 표시 조건
- 모니터링 ON + `monitoring.priceHistory` 존재 + 1건 이상

### 헤더
- `가격 변동 이력 (최근 14일)` (text.muted, semibold)

### SVG 차트 (620x200 viewBox)
- Y축: 원가 (₩), 자동 스케일링 (5단계 그리드)
- X축: 날짜 라벨 (최대 5개)
- 영역: `linearGradient` (primary 18% → 2%)
- 선: primary, 2px, round join/cap
- 데이터 1건: 고정 점 표시 (6px 원, primary 테두리)
- 2건 이상: 호버 시 해당 데이터 포인트에 점 표시

### 호버 인터랙션
- 마우스 X 좌표로 가장 가까운 데이터 포인트 계산
- 해당 열에 `primary` 6% 하이라이트 밴드 표시
- 포인트 원 색상: 수익 <= 0 또는 품절 → `danger`, 그 외 → `primary`

### 호버 툴팁
- 날짜: `formatTooltipDate(entry.date)` (text.secondary, semibold)
- 원가: `₩{sourcePriceKrw}` (primary, bold)
- 수익: `수익 {+/-}₩{profit}` (양수: success, 음수: danger)
- 품절 표시: `entry.stockStatus === 'out_of_stock'`이면 `품절` (danger)

### 타임라인 리스트 (최근 5일)
- 차트 하단에 최근 5개 데이터 역순 표시

| 컬럼 | 내용 |
|------|------|
| 도트 (8px) | 품절: `text.primary`, 역마진: `danger`, 정상: `success` |
| 원가 | `₩{sourcePriceKrw}` + 품절이면 `품절` 태그 |
| 변동 | 기준가 대비 차액. 상승: ▲ danger, 하락: ▼ primary, 변동 없음/품절: `–` |
| 예상 수익 | `{+/-}₩{profit}`. 품절: `–` |
| 날짜 | `formatShortDate(date)` |
