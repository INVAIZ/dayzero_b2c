# [RG-01] 판매 중인 상품 (등록 결과 목록)

## 화면 목적
Qoo10 JP에 등록된 상품을 관리하는 메인 화면.
등록 진행 상태, 판매 성과 요약, 모니터링(가격·품절 변동 확인)을 통합 제공한다.

## 진입 경로
- 사이드바 "판매 중인 상품" 클릭
- ED-01/ED-02에서 "등록하기" 후 이동
- 알림 패널에서 등록 완료/실패 알림 클릭

## 주요 상수
| 상수 | 값 | 용도 |
|------|---|------|
| `FREE_PLAN_LIMIT` | 30 | 무료 플랜 모니터링 최대 건수 |
| `PAGE_SIZE` (AllProductsTable) | 20 | 상품 테이블 페이지당 표시 건수 |

## 섹션 2: 등록 진행 프로그레스 (RegistrationProgressSection)

### 표시 조건
- `jobs` 배열에서 `status === 'processing'`인 작업이 있을 때 표시
- 사용자가 닫기(dismiss)해도 등록은 계속 진행
- 새 등록 작업이 시작되면 dismiss 상태 자동 해제

### 3단계 페이즈 전환

| 페이즈 | 조건 | 배경 | 테두리 | 타이틀 | 서브텍스트 | 퍼센트 색상 |
|--------|------|------|--------|--------|------------|------------|
| `in` (처리 중) | `job.status === 'processing'` | `primaryLight → bg.surface` 그라데이션 | `primaryBorder` | `Qoo10 JP에 등록하고 있어요` | `{currentCount}/{totalCount}건 완료` (currentCount는 파란색 볼드) | `colors.primary` |
| `done` (완료 표시) | `job.status === 'completed'` 후 800ms | `successBg → bg.surface` 그라데이션 | `successBorder` | `등록이 완료되었어요!` | `{totalCount}건 모두 등록 완료` | `colors.success` |
| `out` (퇴장) | done 후 1400ms (= 완료 후 2200ms) | 동일 | 동일 | 동일 | 동일 | 동일 |

### 타이머 시퀀스 (job.status === 'completed' 시점 기준)
1. **+800ms**: phase → `done` (배경색 녹색 전환, 타이틀 변경)
2. **+2200ms**: phase → `out` (퇴장 애니메이션 시작)
3. **+2700ms**: `onDismiss()` 호출 → 카드 제거

### 아이콘 영역 (44x44 박스)
| 페이즈 | 아이콘 |
|--------|--------|
| `in` | Qoo10 로고(`/logos/큐텐.png`, 18px) + Loader2 스피너(14px, 우하단, `spin` 클래스) |
| `done` / `out` | CheckCircle2 (22px, 녹색) |

### 프로그레스 바
- 높이: 6px, 둥근 모서리
- 마운트 시 0%에서 시작 → `requestAnimationFrame` 후 실제 퍼센트로 전이
- 전이: `width 0.8s cubic-bezier(0.16, 1, 0.3, 1)`
- 처리 중: `linear-gradient(90deg, primary, #60A5FA)` + shimmer 오버레이
- 완료: `linear-gradient(90deg, success, successAlt)`, shimmer 제거

### "방금 완료된 상품" 알림
| 조건 | 동작 |
|------|------|
| `job.currentCount`가 증가할 때 | 마지막 결과의 `product.titleKo` 표시 (CheckCircle2 녹색 아이콘 + 상품명) |
| 표시 시간 | 2500ms 후 자동 숨김 |
| 완료 상태일 때 | 숨김 |

### 등록 실패 표시 (부분 실패)

등록 작업이 완료(`job.status === 'completed'`)된 후, 실패 건이 있을 경우:

**프로그레스 카드 done 페이즈 변경:**
- 전체 성공: 기존 동작 (녹색 `등록이 완료되었어요!`)
- 부분 실패 (`failedResults.length > 0`):
  - 배경: `warningLight → bg.surface` 그라데이션, 테두리: `warningBorder`
  - 타이틀: `{totalCount}건 중 {failedCount}건 등록에 실패했어요`
  - 서브텍스트: `성공 {successCount}건 · 실패 {failedCount}건`
  - 아이콘: AlertTriangle (22px, `colors.warningIcon`)

**실패 상품 배너 (프로그레스 카드 아래, 상시 표시):**
- 표시 조건: `failedResults.length > 0`
- 배경: `colors.dangerBg`, 테두리: `colors.dangerLight`, `radius.lg`
- 좌측: AlertCircle (18px, `colors.danger`)
- 텍스트: `등록에 실패한 상품이 {failedCount}건 있어요. 상품을 수정하고 다시 등록해주세요.`
- 우측: `실패 상품 보기` 버튼 (클릭 → "등록 실패" 탭으로 전환)

**등록 실패 에러 유형:**

| 에러 타입 | 사용자 표시 텍스트 |
|-----------|------------------|
| `api_error` | `Qoo10 API 연동 오류` |
| `missing_required` | `필수 항목 누락` |
| `category_mismatch` | `카테고리 불일치` |
| `image_spec` | `이미지 규격 미달` |
| `policy_violation` | `정책 위반` |
| `server_timeout` | `서버 타임아웃 — 잠시 후 다시 시도해주세요` |

### 애니메이션
| 이름 | 설명 |
|------|------|
| `progressCardIn` | `opacity: 0, translateY(16px), scale(0.98)` → `1, 0, 1` (0.5s) |
| `progressCardOut` | `opacity: 1, translateY(0), scale(1)` → `0, -12px, 0.97` (0.5s, forwards) |
| `itemSlideIn` | `opacity: 0, translateY(8px)` → `1, 0` (0.3s) — 방금 완료 상품 |
| `shimmer` | `translateX(-100%)` → `200%` (1.5s, infinite) — 프로그레스 바 반짝임 |

## 섹션 3: 성과 요약 카드 (SuccessSummaryCard)

### 표시 조건
- 프로그레스 카드(`progressJob`)가 없을 때만 표시
- 프로그레스 카드와 상호 배타적

### 3열 그리드 구성

| 카드 | 라벨 | 아이콘 | 값 | 하단 비교 |
|------|------|--------|---|-----------|
| 1 | `판매 중인 상품 수` | Package (18px, text.muted) | `{count}` (salesStatus !== 'paused'인 상품 수) | `+{diff}건` 지난 달 대비 |
| 2 | `건당 평균 예상 수익` | TrendingUp (18px, text.muted) | `₩{avgProfit}` (판매 중 상품의 calcExpectedProfit 평균) | `+₩{diff}` 지난 달 대비 |
| 3 | `총 예상 월 매출` | Coins (18px, text.muted) | `¥{totalRevenue}` + 부가: `₩{totalRevenue * EXCHANGE_RATE}` | `+{pct}%` 지난 달 대비 |

### 하단 비교 텍스트 색상
| 조건 | 색상 |
|------|------|
| 양수 (개선) | `colors.success` |
| 음수 (악화) | `colors.danger` |
| 상품 0건 | 비교 텍스트 숨김 |

### 계산 로직
- `count`: `salesStatus !== 'paused'`인 결과만 카운트
- `totalRevenue`: 위 결과들의 `salePriceJpy` 합산
- `profitSum`: 위 결과들의 `calcExpectedProfit(salePriceJpy, marginRate)` 합산
- `avgProfit`: `profitSum / count` (count > 0일 때)
- 더미 지난 달 대비: `prevProductCount = max(1, round(count * 0.75))`, `prevProfit = max(500, avgProfit - 1200)`, `prevRevenue = round(totalRevenue * 0.8)`

### 애니메이션
- `summaryFadeIn`: `opacity: 0, translateY(12px)` → `1, 0` (0.5s)

## 섹션 5: 모니터링 상태 탭 (MonitoringStatusTabs)

### 7개 탭 정의

| 탭 라벨 | 필터 조건 | 뱃지 variant | 뱃지 표시 조건 |
|---------|-----------|-------------|---------------|
| `전체` | 모든 성공 등록 상품 | `blue` | `counts.total > 0` |
| `판매 중` | `salesStatus !== 'paused'` | `blue` | `counts.active > 0` |
| `가격·품절 확인 중` | `monitoring?.status === 'active'` | `blue` | `counts.monitoring > 0` |
| `품절` | `monitoring?.status === 'active' && lastCheckResult === 'out_of_stock'` | `blue` | `counts.outOfStock > 0` |
| `문제 발생` | `isActualNegativeMargin(r)` (monitoring.lastCheckResult === 'negative_margin' + 실제 마진 검증) | `red` | `counts.negativeMargin > 0` |
| `일시 중지` | `salesStatus === 'paused'` | `gray` | `counts.paused > 0` |
| `등록 실패` | `result.status === 'failed'` | `red` | `counts.failed > 0` |

### 뱃지 색상 규칙

| variant | 활성 탭 | 비활성 탭 |
|---------|---------|----------|
| `blue` | 텍스트: `primary`, 배경: `primaryLight` | 텍스트: `text.muted`, 배경: `bg.subtle` |
| `red` | 텍스트: `danger`, 배경: `dangerLight` | (동일) |
| `gray` | 텍스트: `text.muted`, 배경: `bg.subtle` | (동일) |

### 탭 전환 동작
- 탭 변경 시: 소싱처 필터 → '전체', 검색어 → '', 선택 → 초기화

## 섹션 7: 상품 테이블 (AllProductsTable)

### 컬럼 구성

| 컬럼 | 너비 | 내용 | 비고 |
|------|------|------|------|
| 체크박스 | 20px | 선택 체크박스 | 선택 모드일 때만 |
| 이미지 | 48px | 상품 썸네일 (48x48) | `radius.img` |
| 상태 | 64px | 판매 상태 뱃지 | 아래 참조 |
| 상품명 | flex 3 | 소싱처 로고 + 상품명 | 호버 시 한국어 원문 툴팁 |
| 판매가 | 72px | `¥{salePriceJpy}` | 호버 시 한화 환산 툴팁 |
| 예상 수익 | 72px | `+{profit}원` / `-{profit}원` | 양수: success, 음수: danger |
| 가격·품절 확인 | 100px | 모니터링 토글 (showMonitoring=true) | marginLeft: spacing[6] |
| 등록일 | 64px | `formatShortDate(registeredAt)` | 정렬 가능 (asc/desc/null) |
| 판매처 | 36px | ExternalLink → Qoo10 상품 URL | 없으면 `—` |
| 국내 쇼핑몰 | 70px | ExternalLink → 소싱처 URL | 없으면 `—` |

### 컬럼 헤더
- 선택 모드 + 1건 이상 선택 시: "상품명" 대신 `{selectedIds.length}건 선택` 표시
- 등록일 헤더: 클릭 시 정렬 토글 (desc → asc → null 순환), ChevronUp/ChevronDown 화살표 opacity로 방향 표시

### 판매 상태 뱃지

| salesStatus | 텍스트 | 도트 색상 | 텍스트 색상 | 배경 |

### 행 배경색

| 조건 | 배경 | 호버 배경 |
|------|------|-----------|
| 선택됨 | `colors.primaryLight` | (변경 안 함) |

### 상품명 툴팁 (일본어 번역 있는 경우)
- 조건: `product.titleJa`가 존재

- 툴팁 내용: `한국어 원문` 라벨 + `stripPrefix(product.titleKo)`

### 판매가 툴팁
- 호버 시 FloatingTooltip 표시
- 내용: `한화 환산 (¥1 = ₩{EXCHANGE_RATE})` + `약 ₩{Math.round(salePriceJpy * EXCHANGE_RATE)}`

#### 토글 호버 툴팁
