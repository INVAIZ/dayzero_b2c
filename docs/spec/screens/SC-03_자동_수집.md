# [SC-03] 자동 수집 설정

## 화면 목적
자동 수집 스케줄을 새로 생성하거나 기존 스케줄을 편집한다.
신규 모드는 3단계 위저드 (쇼핑몰 → 카테고리/기준 → 수집 시간), 편집 모드는 올인원 폼.

## 진입 경로
- SC-01 `추가하기` 또는 `+ 자동 수집 추가` 클릭 → 신규 모드 (`/sourcing/auto`)
- SC-01 스케줄 카드 클릭 → 편집 모드 (`/sourcing/auto?edit={scheduleId}`)

## 레이아웃
MainLayout — 최대 너비 800px, 가운데 정렬, 하단 패딩 60px.
페이지 진입 시 `fadeInUp 0.4s ease` 애니메이션 적용.

```
┌──────────────────────────────────────┐
│ ← 뒤로가기                           │
│                                      │
│ 자동 수집 생성 / 자동 수집 수정         │
│ (설명 텍스트)                         │
│                                      │
│ [Step 1 ●]──[Step 2 ○]──[Step 3 ○]  │
│  쇼핑몰      카테고리     수집 시간    │
│                                      │
│ ── (스텝별 콘텐츠) ──                 │
└──────────────────────────────────────┘
```

## 모드 분기

### 신규 모드 (`isEditMode === false`)
- 페이지 제목: `자동 수집 생성`
- 페이지 설명: `원하는 쇼핑몰과 카테고리를 설정하면 매일 자동으로 상품을 수집해요.`
- 스텝 인디케이터 표시
- 3단계 위저드로 진행

### 편집 모드 (`isEditMode === true`, URL에 `?edit={scheduleId}`)
- 페이지 제목: `자동 수집 수정`
- 페이지 설명: `카테고리와 수집 시간을 수정할 수 있어요.`
- 스텝 인디케이터 숨김
- 올인원 폼 (카테고리/기준 + 시간 + 버튼을 한 화면에)
- 기존 스케줄 값 자동 로드: `schedules.find(s => s.id === editId)` → `provider`, `categoryPath`, `criteria`, `timeString`

## 스텝 인디케이터 (신규 모드 전용)

3개 스텝: `쇼핑몰`, `카테고리`, `수집 시간`

### 각 스텝 원의 상태

| 상태 | 배경 | 테두리 | 색상 | 내용 | 추가 효과 |
|------|------|--------|------|------|-----------|
| 완료 (`step.num < currentStep`) | `colors.primary` | 없음 | `colors.bg.surface` | Check 아이콘 (13px, strokeWidth 3) | 클릭 가능 (해당 스텝으로 이동) |
| 현재 (`step.num === currentStep`) | `colors.bg.surface` | `2px solid colors.primary` | `colors.primary` | 스텝 번호 | `subtlePulse` 애니메이션 + `boxShadow: 0 0 0 3px colors.primaryHover` |
| 미완료 (`step.num > currentStep`) | `colors.bg.subtle` | `1.5px solid colors.border.default` | `colors.text.muted` | 스텝 번호 | 클릭 불가 |

### 스텝 라벨

| 상태 | 색상 | fontWeight |
|------|------|------------|
| 완료 또는 현재 | `colors.text.primary` | 현재: `semibold`, 완료: `medium` |
| 미완료 | `colors.text.muted` | `medium` |

### 커넥터 라인 (스텝 사이)
- 너비 56px, 높이 2px
- 배경: `colors.border.default`
- 완료 시 오버레이: `colors.primary`, `width: 100%`, `transition: width 0.6s cubic-bezier(0.16, 1, 0.3, 1)`

### subtlePulse 애니메이션
```css
@keyframes subtlePulse {
    0%, 100% { box-shadow: 0 0 0 3px {colors.primaryHover}; }
    50% { box-shadow: 0 0 0 5px {colors.primaryHover}; }
}
/* 주기: 2.5s ease-in-out infinite */
```

### 스텝 클릭 규칙
- **완료된 스텝만 클릭 가능** (`step.num < currentStep`)
- 미래 스텝 클릭 불가
- 현재 스텝 클릭 불가

#### 시간 드롭다운
- 트리거 버튼: `매일 {formatTimeDisplay(selectedTime)}` (예: `매일 오전 7시`)
  - 배경: `colors.bg.subtle`, 테두리: 기본 `colors.border.default`, 열림 시 `colors.primary`

- `hour < 12` → `오전 {hour}시`
- `hour === 12` → `오후 12시`
- `hour > 12` → `오후 {hour - 12}시`

- 06:00 ~ 23:00 (18개 옵션, 1시간 단위)

## 저장 동작

### 신규 저장 (`handleSave`, editId 없음)

- **Given**: Step 3, 모든 조건 설정됨
- **When**: `설정 저장하기` 클릭
- **Then**:
  1. `addSchedule()` 호출:
     - `id: 'sched-{timestamp}-{random}'`
     - `provider`, `categoryPath`, `criteria`, `timeString`
     - `targetCount: 50`, `isActive: true`
  2. 알림 생성:
     - `type: 'auto'`
     - `title: '{쇼핑몰} {카테고리} 자동 수집이 등록되었어요'`
     - `status: 'scheduled'`
     - `totalCount: 50`
     - `timeString: selectedTime`
  3. `setSelectedAutoFilter('전체')` — SC-01의 자동 수집 필터 초기화
  4. `/sourcing` (SC-01)로 이동

### 편집 저장 (`handleSave`, editId 있음)

- **Given**: 편집 모드, 설정 변경됨
- **When**: `수정 저장하기` 클릭
- **Then**:
  1. `updateSchedule(editId, { provider, categoryPath, criteria, timeString })` 호출
  2. 알림 생성 없음
  3. `/sourcing` (SC-01)로 이동
