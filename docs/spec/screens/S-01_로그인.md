# [S-01] 로그인

## 화면 목적
서비스 진입점. 이메일 로그인/회원가입 또는 Google 소셜 로그인을 제공한다.

## 진입 경로
- 앱 최초 접속 (`/`)
- 로그아웃 후 리다이렉트

## 소스 파일
- `pages/auth/LoginPage.tsx` — 페이지 컨테이너, 모드 전환, 토스트 관리
- `components/auth/LoginForm.tsx` — 로그인 폼
- `components/auth/SignupForm.tsx` — 회원가입 폼
- `components/auth/PasswordInput.tsx` — 비밀번호 입력 공통 컴포넌트
- `components/auth/SocialLoginButton.tsx` — Google 소셜 로그인 버튼
- `hooks/useAuthForm.ts` — 이메일/비밀번호 검증 로직
- `mock/authMock.ts` — 더미 API (로그인 1.2s, 회원가입 1.5s, Google 0.8s)

## 상태 관리

| 상태 | 타입 | 초기값 | 설명 |
|------|------|--------|------|
| `mode` | `'login' \| 'signup'` | `'login'` | 현재 폼 모드 |
| `formKey` | `number` | `0` | 모드 전환 시 증가하여 폼 재렌더링 |
| `transitioning` | `boolean` | `false` | 모드 전환 애니메이션 진행 중 |
| `toasts` | `Toast[]` | `[]` | 활성 토스트 목록 |

## 모드 전환 애니메이션

1. `switchMode()` 호출 시 `transitioning`이 `true`이면 무시 (이중 클릭 방지)
2. `transitioning = true` → 현재 폼이 **0.18초** 동안 페이드아웃
   - `opacity: 0`, `transform: translateY(6px)`
   - `transition: opacity 0.18s ease, transform 0.18s ease`
3. **180ms** 후 (`setTimeout`):
   - `mode` 변경
   - `formKey` 증가 (React key 변경으로 폼 리마운트)
   - `transitioning = false` → 새 폼이 **0.18초** 동안 페이드인
   - `opacity: 1`, `transform: translateY(0)`

LoginPage 자체 구현. 글로벌 토스트(G-03)와 별도.

**위치:** 화면 상단 중앙 (`position: fixed`, `top: 16px`, `left: 50%`, `transform: translateX(-50%)`)

**구조:**
```
┌──────────────────────────────┐
│▌ 토스트 메시지 텍스트          │
└──────────────────────────────┘
 ↑ 좌측 색상 바 (4px)
```

**색상 바 (좌측 4px):**

| 타입 | 색상 |
|------|------|
| `success` | `colors.successAlt` |
| `error` | `colors.danger` |
| `info` | `colors.primary` |

**스타일:**
- 배경: `colors.bg.surface`
- 모서리: `radius.lg`
- 그림자: `0 4px 20px rgba(0,0,0,0.14)`
- 텍스트: `font.size.md`, `font.weight.medium`, `colors.text.primary`
- 자동 소멸: **3초** 후
- 여러 토스트 동시 가능 (세로 배치, gap `8px`)

## 로그인 모드 (기본)

### 구성 요소 (위→아래 순서)

1. **DayZero 로고** — 중앙 정렬, 높이 `40px`, `marginBottom: 40px`
2. **이메일 필드**
3. **비밀번호 필드**
4. **"비밀번호를 잊으셨나요?"** 링크 — 우측 정렬
5. **"로그인" CTA 버튼**
6. **회원가입 유도 블록** — `colors.bg.subtle` 배경, `radius.lg`
7. **구분선** — "또는" 텍스트
8. **Google 로그인** — 원형 버튼 (48x48)

### 이메일 필드

**라벨:** "이메일" (`font.size.md`, `font.weight.semibold`, `colors.text.primary`)

**입력:**
- `type="email"`
- placeholder: `"이메일을 입력해주세요"`
- 배경: `colors.bg.subtle`, `radius.lg`
- 테두리 기본: `1.5px solid transparent`
- `font.size.base`, `colors.text.primary`
- 공백 자동 제거 (`value.replace(/\s/g, '')`)

**포커스/블러 스타일 변화:**

| 상태 | 테두리 색상 | 전환 |
|------|------------|------|
| 기본 | `transparent` | — |
| 포커스 (에러 없음) | `colors.primary` | `0.18s ease` |
| 블러 (에러 없음) | `transparent` | `0.18s ease` |
| 에러 있음 | `colors.danger` | `0.18s ease` |

**검증 (`useAuthForm.validateEmail`):**
- 타이밍: `onBlur` 시 검증 실행, 에러 있는 상태에서 타이핑 시 실시간 재검증
- 규칙 1: 빈 값 → `"이메일을 입력해주세요"`
- 규칙 2: `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` 불일치 → `"올바른 이메일 형식이 아니에요. @ 기호를 포함해주세요"`
- 통과 시: `null` (에러 없음)

**에러 표시:** 필드 하단 `marginTop: 6px`, `font.size.sm`, `colors.danger`

### 비밀번호 필드

**라벨:** "비밀번호" (`font.size.md`, `font.weight.semibold`, `colors.text.primary`)

**입력 (PasswordInput 컴포넌트, `variant="rounded"`):**
- `type`: `password` (기본) / `text` (표시 토글)
- placeholder: `"비밀번호를 입력해주세요"`
- padding: `14px 40px 14px 16px` (우측에 토글 버튼 공간)
- 배경: `colors.bg.subtle`, `radius.lg`
- 테두리: 에러 시 `1.5px solid colors.danger`, 기본 `1.5px solid transparent`

**표시/숨김 토글:**
- 위치: 필드 우측 (`right: 12px`)
- 아이콘: `Eye` (18px) / `EyeOff` (18px)
- 색상: 기본 `colors.text.placeholder`, 호버 `colors.text.muted`
- 전환: `0.15s`

**검증:** 제출 시에만 검증
- 빈 값 → `"비밀번호를 입력해주세요"`
- 타이핑 시 에러 클리어

### "비밀번호를 잊으셨나요?" 링크

- 위치: 비밀번호 필드 아래 우측 정렬, `marginBottom: 28px`
- 스타일: `font.size.sm`, `colors.text.disabled`, `btn-link` 클래스
- 클릭 시: 토스트 표시 (아래 참조)

### 로그인 CTA 버튼

- 클래스: `btn-primary` (전역 CSS)
- 텍스트: `"로그인"` (기본) / `"로그인 확인 중..."` (로딩)
- 로딩 시: `Loader2` 스피너 (18px, `spin 1s linear infinite`) + 텍스트
- `disabled`: 로딩 중

### 회원가입 유도 블록

- 배경: `colors.bg.subtle`, `radius.lg`, padding `14px`, 중앙 정렬
- 텍스트: `"DayZero가 처음이에요"` (`font.size.md`, `colors.text.muted`)
  - "DayZero"는 `JetBrains Mono, monospace`, `font.weight.semibold`
- 링크: `"가입하기"` (`colors.primary`, `font.weight.bold`, `font.size.md`)

### 구분선

- 구조: 좌측 1px 선 + "또는" 텍스트 + 우측 1px 선
- 선 색상: `colors.border.default`
- 텍스트: `font.size.sm`, `colors.text.placeholder`
- 여백: `margin: 24px 0 20px`

### Google 로그인 버튼 (로그인 모드)

- 원형 버튼 48x48, `border: 1.5px solid colors.border.default`
- 배경: `colors.bg.surface`
- 내부: Google "G" SVG 아이콘 (20x20, 4색)
- 호버: 배경 `colors.bg.faint`, 테두리 `colors.border.light`
- 전환: `background 0.15s, border-color 0.15s, transform 0.1s`

## 회원가입 모드

### 구성 요소 (위→아래 순서)

1. **DayZero 로고** — 중앙, 높이 `30px` (로그인보다 작음), `marginBottom: 36px`
2. **Google 소셜 로그인 버튼** (SocialLoginButton 컴포넌트)
3. **구분선** — "또는"
4. **이메일 필드** (밑줄 스타일 `input-line`)
5. **비밀번호 필드** (밑줄 스타일, `marginTop: 20px`)
6. **비밀번호 조건 체크리스트** — 비밀번호 1자 이상 입력 시 표시
7. **비밀번호 확인 필드** (`marginTop: 20px`, `marginBottom: 28px`)
8. **"가입하기" CTA 버튼**
9. **로그인 유도 링크**

### Google 소셜 로그인 버튼 (SocialLoginButton)

- 클래스: `btn-google` (전역 CSS)
- 내부: Google "G" SVG (18x18) + 텍스트
- 텍스트: `"Google로 시작하기"` (기본) / `"연결 중..."` (로딩)
- `disabled`: 로딩 중

### 이메일 필드 (회원가입)

- 클래스: `input-line` (밑줄 스타일, 로그인 모드와 다름)
- placeholder: `"이메일"`
- 에러 시: 클래스에 `has-error` 추가
- 검증: 로그인 모드와 동일 (`onBlur` 시 + 에러 상태에서 실시간)

### 비밀번호 필드 (회원가입)

- PasswordInput 컴포넌트, `variant="line"` (기본값)
- placeholder: `"비밀번호를 입력해주세요"` (기본값)
- 클래스: `input-line pw-input`

### 비밀번호 조건 체크리스트

**표시 조건:** `password.length > 0` (1자 이상 입력 시)

**레이아웃:** 세로 배치, gap `6px`, `marginTop: 12px`

| 조건 키 | 라벨 텍스트 | 검증 규칙 |
|---------|-----------|-----------|
| `length` | `8자 이상` | `password.length >= 8` |
| `letter` | `영문 포함` | `/[a-zA-Z]/.test(password)` |
| `number` | `숫자 포함` | `/[0-9]/.test(password)` |
| `special` | `특수문자 포함 (!@#$%^&*)` | `/[!@#$%^&*]/.test(password)` |

**각 항목 스타일:**

| 상태 | 아이콘 | 아이콘 색상 | 텍스트 색상 |
|------|--------|-----------|-----------|
| 충족 | `CheckCircle2` (14px) | `colors.successAlt` | `colors.successAlt` |
| 미충족 | `Circle` (14px) | `colors.border.light` | `colors.text.disabled` |

텍스트: `font.size.sm`

**실시간 업데이트:** 비밀번호 타이핑할 때마다 `getPasswordChecks(password)` 재실행.

### 비밀번호 확인 필드

- PasswordInput, placeholder: `"비밀번호 확인"`
- 에러 전달: `error` prop으로 에러 메시지 표시
- 검증 타이밍:
  - `onChange` 시: 에러 상태에서만 재검증 (`confirmError`가 있을 때)
  - 제출 시: 항상 검증
- 에러 메시지: `"비밀번호가 일치하지 않아요"`

### 가입하기 CTA 버튼

- 클래스: `btn-primary`
- 텍스트: `"가입하기"` (기본) / `"가입 처리 중..."` (로딩)
- 로딩 시: `Loader2` 스피너 (18px) + 텍스트
- `disabled`: 로딩 중

**제출 검증 순서:**
1. 이메일 검증 (`validateEmail`)
2. 비밀번호 4개 조건 모두 충족 (`isPasswordValid`)
3. 비밀번호 확인 일치
4. 하나라도 실패 시 해당 에러 표시, API 호출 안 함

**성공 흐름:**
1. `mockSignup()` 호출 (1.5초 대기)
2. 토스트: `"가입 완료! DayZero를 시작할게요"` (type: `success`)
3. **1.5초** 후 `onSuccess()` → `/qoo10-connect` 이동

### 로그인 유도 링크

- 텍스트: `"이미 계정이 있으신가요?"` + `"로그인하기"` 링크
- `font.size.md`, `colors.text.muted`
- 링크: `colors.primary`, `font.weight.semibold`

## 동작 정의

### 이메일 로그인
- Given: 로그인 모드, 이메일과 비밀번호 입력됨
- When: "로그인" 버튼 클릭
- Then: 이메일/비밀번호 검증 → 실패 시 에러 표시, 성공 시:
  1. `loading = true`, 버튼 "로그인 확인 중..." + 스피너
  2. `mockLogin()` (1.2초 대기)
  3. `loading = false`
  4. `onSuccess()` → `/qoo10-connect` 즉시 이동

### Google 로그인 (양쪽 모드)
- Given: 아무 모드
- When: Google 버튼 클릭
- Then:
  1. `googleLoading = true`
  2. `mockGoogleLogin()` (0.8초 대기)
  3. `googleLoading = false`
  4. `onSuccess()` → `/qoo10-connect` 즉시 이동

### 비밀번호 찾기
- Given: 로그인 모드
- When: "비밀번호를 잊으셨나요?" 클릭
- Then: 토스트 `"입력하신 이메일로 재설정 링크를 보냈어요. 메일함을 확인해주세요"` (type: `info`)

## 검증 체크리스트
- [ ] 로그인 모드: 이메일/비밀번호 입력 → 로그인 성공 → S-02 이동
- [ ] 회원가입 모드: 모든 필드 입력 → 회원가입 성공 → 토스트 → 1.5초 후 S-02 이동
- [ ] Google 로그인 → S-02 이동 (로그인 모드에서 원형 버튼, 회원가입 모드에서 풀 버튼)
- [ ] 이메일 검증: 빈값 에러 + 형식 에러 (2종)
- [ ] 이메일 블러 시 검증 실행, 에러 상태에서 타이핑 시 실시간 재검증
- [ ] 비밀번호 조건 체크리스트: 4개 항목 실시간 업데이트 (1자 이상 입력 시 표시)
- [ ] 비밀번호 확인 불일치 에러
- [ ] 모드 전환 애니메이션: 0.18초 페이드아웃 → 폼 교체 → 0.18초 페이드인
- [ ] 전환 중 재클릭 방지 (`transitioning` 가드)
- [ ] 비밀번호 찾기 → info 토스트 (파란 좌측 바)
- [ ] 로딩 중 버튼 disabled + 스피너
- [ ] 토스트 3종 색상 바: success(초록) / error(빨강) / info(파랑)
- [ ] 토스트 3초 후 자동 소멸
- [ ] 비밀번호 표시/숨김 토글 (Eye/EyeOff)
- [ ] 인풋 포커스 시 테두리 파란색, 블러 시 투명 복귀
