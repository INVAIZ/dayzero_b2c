# 공통 UI 패턴

> 모든 화면에서 반복적으로 사용되는 글로벌 UI 요소를 정의합니다.
> 화면별 명세에서 "→ 공통 패턴 참조"로 표기된 항목은 이 문서를 기준으로 합니다.

---

## G-01. 사이드바 (Sidebar)

### 개요
온보딩 완료 후 모든 메인 화면 좌측에 고정 표시되는 네비게이션.
온보딩 화면(S-01~S-05)에서는 표시되지 않습니다.

### 구조

```
┌─────────────────────────┐
│                         │
│  [DayZero 로고]          │  ← 클릭 시 /sourcing 이동
│                         │
│  🔍 수집하기             │  ← /sourcing
│  📋 수집된 상품 (N)      │  ← /editing (뱃지: 미처리 수)
│  📦 판매 중인 상품        │  ← /registration
│                         │
│  ─────── 구분선 ──────── │
│                         │
│  ⚙️ 설정          ▾     │  ← 클릭 시 하위 메뉴 토글
│     ├─ 판매 설정         │  ← /settings/sales
│     ├─ Qoo10 연동        │  ← /settings/qoo10
│     └─ 계정              │  ← /settings/account
│                         │
│        (여백)            │
│                         │
└─────────────────────────┘
```

### 스타일
- 너비: 280px, 높이: 100vh, 좌측 fixed
- 배경: `colors.bg.page` (#F9FAFB)
- 우측 테두리: `colors.border.default` 1px
- 패딩: 상하 `spacing['10']`, 좌우 `spacing['6']`
- 로고 영역: 하단 마진 48px

### 메뉴 아이템
- 아이콘: lucide-react (`Search`, `LayoutList`, `PackageOpen`, `Settings`)
- 아이콘 크기: 20px
- 폰트: `font.size.base`, `font.weight.medium`
- 간격: 아이콘과 텍스트 사이 `spacing['3']`
- 메뉴 간 간격: `spacing['2']`
- 패딩: `spacing['3']`
- border-radius: `radius.lg`

### 활성 상태 판정

| 메뉴 | 활성 조건 | 활성 스타일 |
|------|-----------|------------|
| 수집하기 | pathname.startsWith('/sourcing') | 배경: `colors.border.default`, 텍스트: `colors.text.primary` |
| 수집된 상품 | pathname.startsWith('/editing') | 동일 |
| 판매 중인 상품 | pathname.startsWith('/registration') | 동일 |
| 설정 하위 항목 | pathname === `/settings/{key}` | 동일 |

- **비활성**: 배경 transparent, 텍스트 `colors.text.tertiary`, 아이콘 `colors.text.muted`
- **호버**: 배경 `colors.bg.subtle`

### 뱃지 (수집된 상품)
- 조건: `unprocessedProductCount > 0` 이고 해당 메뉴가 비활성일 때
- 위치: 메뉴 텍스트 우측 (`marginLeft: auto`)
- 스타일: 배경 `colors.primary`, 텍스트 흰색, `radius.img`, `font.size.xs`, `font.weight.bold`
- 애니메이션: 수가 증가하면 bounce 효과 0.6초

### 설정 하위 메뉴
- 설정 버튼에 `ChevronDown` 아이콘 (16px, `colors.text.muted`)
- 펼침 시 아이콘 180도 회전 (transition 0.2s)
- 설정 영역 진입 시(`/settings/*`) 자동 펼침
- 하위 메뉴에 트리 라인 커넥터 (세로 + 가로 브랜치)
  - 세로 라인: left 22px, 너비 1px, `colors.border.default`
  - 가로 브랜치: 각 항목 좌측 10px

## G-02. 알림 패널 (NotificationPanel)

### 개요
판매 중인 상품의 가격 변동/품절/재입고 알림을 표시하는 플로팅 패널.
**우측 하단 FAB(Bell 아이콘)이 유일한 알림 진입점**입니다. 사이드바에는 알림 아이콘이 없습니다.

### 알림 유형

| 유형 | 아이콘 | 아이콘 배경 | 설명 텍스트 |
|------|--------|------------|------------|
| 역마진 (자동 조정됨) | `TrendingUp` | `colors.primaryLight` / `colors.primary` | "원가가 ₩{이전} → ₩{현재}로 올라 판매가가 자동으로 ¥{조정가}로 조정됐어요." |
| 역마진 (미조정) | `AlertTriangle` | `colors.dangerLight` / `colors.danger` | "원가가 ₩{이전} → ₩{현재}로 올라 역마진이 발생했어요. 판매가 조정이 필요해요." |
| 품절 (자동 중지됨) | `PauseCircle` | `colors.orangeLight` / `colors.orange` | "{소싱처}에서 품절이 감지돼 자동으로 판매가 중지됐어요." |
| 품절 (미처리) | `PackageX` | `colors.dangerLight` / `colors.danger` | "{소싱처}에서 품절이 감지됐어요. 판매 일시중지가 필요해요." |
| 재입고 | `PackageCheck` | `colors.successLight` / `colors.success` | "{소싱처}에서 재입고가 확인돼 자동으로 판매가 재개됐어요." |

### FAB (Floating Action Button)
- 위치: 우측 하단 고정 (bottom 24px, right 24px)
- 크기: 56x56px, 원형 (`radius.full`)
- 배경: `colors.text.primary` (진한 색)
- 아이콘: `Bell` 22px, 흰색
- 호버: `colors.primaryDark` (#2563EB)
- 그림자: `shadow.lg`
- 안읽은 알림 뱃지: 우상단 (-4px), `colors.danger` 배경, 흰색 텍스트, 9건 초과 시 "9+"

### 패널 구조

```
┌──────────────────────────────────┐
│ 상품 알림          [모두 읽음] [X] │
├──────────────────────────────────┤
│ 전체 | 가격 변동 | 품절 | 재입고   │  ← 탭 (안읽은 건수 뱃지)
├──────────────────────────────────┤
│ [아이콘] 상품명 (일본어)    3분 전 │
│          설명 텍스트...           │
│──────────────────────────────────│
│ [아이콘] 상품명             어제   │
│          설명 텍스트...           │
│          ...                     │
└──────────────────────────────────┘
```

- 위치: FAB 위 12px (`bottom: calc(100% + 12px)`)
- 너비: 460px
- 배경: `colors.bg.surface`, border `colors.border.default`
- border-radius: `radius.xl`
- 그림자: `shadow.lg`
- 콘텐츠 영역 높이: 520px (스크롤)
- 진입 애니메이션: `modalSlideUp` 0.25s

### 패널 헤더
- 제목: "상품 알림" (`font.size.lg`, `font.weight.bold`)
- "모두 읽음으로 표시" 버튼: 안읽은 알림 있을 때만 표시 (`colors.primary`, `font.size.sm`)
- 닫기(X) 버튼

### 탭
- 전체 / 가격 변동 / 품절 / 재입고
- 활성 탭: 하단 2px 보더 `colors.primary`, 텍스트 `colors.primary`, `font.weight.bold`
- 비활성 탭: 텍스트 `colors.text.tertiary`
- 안읽은 건수 뱃지: 활성 시 `colors.primaryLight`/`colors.primary`, 비활성 시 `colors.bg.subtle`/`colors.text.muted`

### 알림 아이템
- 안읽은 배경: `colors.bg.unread`
- 호버 배경: `colors.bg.faint`
- 클릭 시: 읽음 처리 + 패널 닫기 + `/registration/{resultId}` 이동
- 호버 시: 시간 텍스트 대신 X(dismiss) 버튼 표시
- 각 아이템 진입 애니메이션: `fadeInUp` 0.3s (index * 0.04s 딜레이)

### 빈 상태
- 조건: 필터링된 알림이 0건
- 아이콘: `Bell` 20px in 48px 원형 (`colors.bg.subtle` 배경)
- 텍스트: "가격·품절 확인 중인 상품에서\n변동 사항이 없어요"
- 프로토타입 전용: "변동 시뮬레이션" 버튼 (프로덕션에서는 제거)

### 시간 표시 형식
- 1분 미만: "방금"
- 1~59분: "N분 전"
- 1~23시간: "N시간 전"
- 어제: "어제"
- 그 이전: "MM.DD"

### 상태 관리
- `useNotificationStore`: 읽음(readIds), 삭제(dismissedIds) 관리
- 알림 데이터는 `useRegistrationStore`의 등록 결과에서 monitoring 정보를 추출
- 삭제된 상품 클릭 시: 토스트 "이미 삭제된 상품이에요." + 자동 dismiss

## G-03. 토스트 메시지 (ToastContainer)

### 개요
사용자 액션에 대한 즉각적인 피드백을 제공하는 비침습적 알림.

### 스타일
- 위치: 우측 하단 고정 (bottom 96px, right 24px — FAB 위)
- 배경: `colors.bg.surface`, 테두리 `colors.border.default`
- 그림자: `0 8px 32px rgba(0, 0, 0, 0.12)`
- border-radius: `radius.xl` `radius.xl` `radius.xs` `radius.xl` (우하단만 작게 — 말풍선 꼬리)
- 말풍선 꼬리: 우측 하단 12px 위치, 12x12px 45도 회전
- 최대 너비: 400px
- 폰트: `font.size.base`, `font.weight.semibold`
- 진입 애니메이션: `toastBubbleUp` 0.4s (아래→위, 스케일 0.9→1)

### 구조
- 메인 메시지: `colors.text.primary`, `font.weight.semibold`
- 상세 텍스트 (선택): `colors.text.secondary`, `font.weight.medium`, `font.size.md`
- 여러 토스트가 동시에 뜰 수 있음 (세로 12px 간격)

### 자동 소멸
- 기본 5초 후 자동 제거
- `duration` 파라미터로 개별 조정 가능
- `duration: 0`이면 수동으로만 제거

### 발생하는 토스트 목록

| 화면 | 조건 | 메시지 | 상세 |
|------|------|--------|------|
| ED-01 | AI 번역 시작 | "N건 번역을 시작했어요" | — |
| ED-01 | 상품 삭제 완료 | "N건이 삭제되었어요" | — |
| ED-02 | 저장 완료 | "저장되었어요" | — |
| ED-02 | 번역 완료 | "번역이 완료되었어요" | — |
| RG-01 | 등록 시작 | "N건 등록을 시작했어요" | — |
| RG-02 | 삭제된 상품 알림 클릭 | "이미 삭제된 상품이에요" | — |
| RG-03 | 저장 완료 | "저장되었어요" | — |
| ST-01 | 설정 저장 | "설정이 저장되었어요" | — |
| ST-01 | 가격 재계산 | "{변경 항목}이(가) 변경되어 N건의 판매가가 재계산돼요" | — |

> **토스트 어투 규칙:** 모든 토스트는 `~했어요` / `~되었어요` / `~이에요` (구어체 존댓말)로 통일합니다.

## G-04. 확인 모달 (ConfirmModal)

### 개요
파괴적 액션(삭제 등) 또는 중요 안내 전 사용자 확인을 받는 모달.

### 구조

```
┌──────────────────────────────────┐
│                            [X]  │
│         [아이콘 원형]            │
│                                 │
│           제목 텍스트            │
│         설명 텍스트 ...          │
│                                 │
│    [ 취소 ]     [ 확인/삭제 ]    │
└──────────────────────────────────┘
```

### 스타일
- 백드롭: `rgba(0, 0, 0, 0.4)` + `backdrop-filter: blur(4px)`
- 모달: max-width 440px, 패딩 32px 24px 24px
- border-radius: `radius.xl`
- 그림자: `shadow.lg`
- 진입 애니메이션: 백드롭 `fadeIn` 0.2s, 모달 `slideUp` 0.3s

### 타입별 스타일

| 타입 | 아이콘 배경 | 아이콘 색상 | 확인 버튼 색상 |
|------|------------|------------|--------------|
| `danger` | `colors.dangerBg` | `colors.danger` | `colors.danger` |
| `info` | `colors.primaryLight` | `colors.primary` | `colors.primary` |

### 아이콘
- `AlertCircle` (lucide-react) 28px
- 원형 배경 56x56px

### 버튼
- 2열 배치 (flex, gap 12px, 각각 flex: 1)
- 취소: `colors.bg.subtle` 배경, `colors.text.secondary`, 호버 시 `colors.border.default`
- 확인: 타입별 색상 배경, 흰색 텍스트, 호버 시 opacity 0.9
- 패딩: 14px, `radius.md`, `font.size.md`, `font.weight.semibold`/`bold`
- 기본 텍스트: 취소 "취소", 확인 "삭제하기" (props로 커스텀 가능)

### 동작
- 백드롭 클릭 → 닫기
- X 버튼 클릭 → 닫기
- 확인 클릭 → `onConfirm()` 실행 후 자동 닫기

## G-05. 공통 상태 컴포넌트

### StatusIcon
작업 상태를 아이콘으로 표시하는 컴포넌트.

| status | 아이콘 | 배경 | 아이콘 색상 |
|--------|--------|------|------------|
| `completed` | `CheckCircle2` | `colors.successBg` | `colors.success` |
| `failed` | `X` | `colors.dangerBg` | `colors.danger` |
| `scheduled` | `Zap` (filled) | `colors.scheduledBg` | `colors.scheduledIcon` |
| `running` / `processing` / `queued` | `Loader2` (spin) | `colors.primaryLight` | `colors.primary` |

- 크기: 36x36px, border-radius: `radius.img`

### ProgressBar
작업 진행률 표시 바.

- 높이: 4px
- 배경(트랙): `colors.bg.subtle`
- 진행(바): `colors.primary`
- border-radius: `radius['2xs']`
- 너비 전환: `transition: width 0.4s ease`

### EmptyState
목록에 데이터가 없을 때 표시하는 빈 상태.

- 아이콘: `PackageOpen` 36px, `colors.border.default`
- 텍스트: `font.size.sm`, `colors.text.muted`
- 패딩: 56px 24px, 가운데 정렬

## G-06. 레이아웃 (MainLayout)

### 구조
```
┌──────────┬─────────────────────────────────┐
│          │                                 │
│ Sidebar  │         Main Content            │
│ (280px)  │    (padding: 48px 64px)         │
│ fixed    │    (max-width: 1200px)          │
│          │    (margin: 0 auto)             │
│          │                                 │
└──────────┴─────────────────────────────────┘
```

- 사이드바: 280px 고정, 좌측 fixed (→ G-01 참조)
- 메인 영역: `marginLeft: 280px`, 배경 `colors.bg.surface` (흰색)
- 콘텐츠: `maxWidth: 1200px`, 좌우 중앙 정렬
- 패딩: 48px 64px
- 폰트: `Pretendard, -apple-system, sans-serif`

### 포함 요소
MainLayout은 자동으로 다음을 포함합니다:
- `<Sidebar />` — 좌측 네비게이션
- `<ToastContainer />` — 토스트 메시지
- `<NotificationPanel />` — 알림 패널 (FAB + 패널)

### 적용 화면
- SC-01~SC-03, ED-01, RG-01~RG-02: MainLayout 사용
- ED-02, RG-03, ST-01: Sidebar 직접 포함 (MainLayout 미사용, 자체 레이아웃)
- S-01~S-05: 레이아웃 없음 (전체 화면)
