# DayZero B2C — UT 테스트 기획 베이스 문서

> **작성일**: 2026-03-16
> **목적**: 현재 프로토타입 기준 전체 화면 구조 및 UX 흐름 정리. UT 시나리오 설계 참고용.
> **구현 방식**: 더미 데이터 기반 프로토타입. 실제 API 연동 없음.

---

## 서비스 개요

한국 개인 셀러가 국내 쇼핑몰(올리브영·쿠팡·다이소)에서 상품을 수집해, Qoo10 JP에 자동 번역·등록·판매하는 크로스보더 SaaS.

**핵심 사용자 여정**: 온보딩 → 수집 → 편집 → 등록 → 판매 관리

## 흐름 1: 온보딩

**진입 경로**: `/` (로그인) → `/basic-info` → `/qoo10-connect` → `/basic-margin` → `/sourcing`

### OB-01 기본 정보 입력
- 배송대행사 선택 (큐익스프레스 / 링코스 / KSE / 직접 입력)
- 선택 시 일본 창고 주소 자동 입력
- 반품 주소 (배송지와 동일 체크박스)
- 연락처 입력
- **완료 조건**: 모든 필수 필드 입력 → 다음 버튼 활성화

### OB-02 Qoo10 계정 연결
- API Key 입력 → 연결하기 버튼
- 더미: 임의 키 입력 시 성공 처리 (Mock 연결)
- QSM 설치 안내 패널 (슬라이드 아웃)
- 연결 성공 시 판매자 ID · 스토어명 표시

### OB-03 기본 마진 설정
- 배송대행사별 요율표 표시 (무게 구간별 배송비 ¥)
- 상품 카테고리별 예상 무게 입력
- 마진율 계산기: 원가 입력 → 목표 마진 → 권장 판매가 자동 계산
- 환율 고정값 사용 (9.2원/엔)
- 플랫폼 수수료 12% 반영

## 흐름 2: 수집

**진입 경로**: 사이드바 → 수집 목록 (`/sourcing`)

### SC-01 수집 목록 메인
- **자동 수집 탭 (기본)**:
  - 등록된 스케줄 카드 리스트 (소싱처별)
  - 스케줄 토글 ON/OFF
  - 드래그로 순서 변경
  - 스케줄 삭제
  - 신규 스케줄 추가 버튼 → SC-03으로 이동
- **URL 수집 탭**:
  - URL 입력창 인라인 렌더링 (SC-01 안에 임베드)
  - → SC-02 흐름과 동일하게 동작

### SC-02 URL 수집 실행 (`/sourcing/url`)
1. URL 텍스트 입력 (줄바꿈으로 다수 입력, 최대 20개)
2. 입력 즉시 유효성 파싱: 소싱처 자동 감지 (올리브영/쿠팡/다이소)
3. 수집 시작 버튼 → 각 URL 순차 처리 (2.5~5초 딜레이 시뮬레이션)
4. 진행 상태: idle → running → completed (UT에서는 항상 성공)
5. 전체 완료 → "수집 완료" 토스트 + "수집 목록 보기" 버튼
6. `/editing`으로 이동

**특이사항**: UT 목적으로 수집 실패 케이스 제거 (항상 성공 처리)

## 흐름 3: 편집

**진입 경로**: 사이드바 → 수집 목록 (`/editing`) 또는 수집 완료 후 자동 이동

### ED-01 수집 목록 (편집 대기 리스트)

**상단**:
- 페이지 헤더 + 수집 기록 모달 버튼
- 수집 진행 중 배너 (진행 중일 때만)
- 소싱처 필터 탭 (전체 / 올리브영 / 쿠팡 / 다이소)
- 탭 필터: 전체 / 편집 필요 / 편집 완료

**상품 목록 (ProductListItem)**:
- 썸네일, 소싱처 로고, 상품명(한국어), 판매가(¥), 마진율
- **편집 필요 태그**: `⚠ 편집 필요 · 상품명 / 옵션 / 상세설명` (미완료 항목만 표시)
  - 호버 시 상세 툴팁: "등록하기 전에 편집이 필요해요." + 3개 체크리스트 (상품명 번역 / 옵션 번역 / 상세설명 작성 및 번역)
  - 완료 항목: 초록 체크, 미완료: 주황 AlertCircle
- 정렬: 등록일 / 판매가 / 상품명 (오름차순/내림차순)
- 검색: 상품명으로 필터

**체크박스 선택 + 벌크 액션바**:
- AI로 편집하기 (미번역 상품 수) — Sparkles 아이콘
- Qoo10에 등록하기 (번역 완료 상품 수)
- 삭제하기

### ED-02 편집 상세 (`/editing/:productId`)

**헤더**: 뒤로가기, 이전/다음 상품 내비게이션, Qoo10 등록 버튼

**4개 탭**:

#### 탭 1: 기본 정보
- **상품명 (일본어)**:
  - 번역 전: `* 상품명` 경고 아이콘 + 힌트 텍스트
  - 한국어 입력 시: "번역이 필요해요" 힌트 + AI 번역 버튼 활성화
  - 번역 완료: `✓ 상품명` 체크 아이콘
- **옵션 (일본어)**:
  - 옵션 카드 리스트 (이름 / 추가금액)
  - 미번역 시 경고 아이콘 + 힌트
  - 전체 옵션 번역 완료 시 체크 아이콘
- **상세설명 (일본어)**:
  - 텍스트에리어
  - 미작성/한국어 시: 경고 아이콘 + "일본어 상세설명을 작성해 주세요. 어렵다면 AI 작성 버튼을 활용해 보세요."
  - 작성 완료: 체크 아이콘
  - AI 작성 버튼 (더미 딜레이 후 초안 생성)

#### 탭 2: 가격 정보
- 원가(₩), 판매가(¥), 마진율 실시간 계산
- 배송비 설정

#### 탭 3: 썸네일
- 대표 이미지 선택 / 순서 변경

#### 탭 4: 상세 이미지
- 상세 이미지 리스트 관리

## 흐름 4: 등록 및 판매 관리

### RG-01 판매 중인 상품 (`/registration`)

**상단 헤더**:
- "판매 중인 상품" + Qoo10 JP 뱃지
- 등록 기록 버튼 (완료된 Job 있을 때)
- 변동 기록 버튼 (변동 알림 등록 상품 있을 때)

**대시보드 (항상 표시)**:
4개 요약 카드:
- 등록된 상품 수 (상품 있을 때만 지난 달 대비 표시)
- 평균 마진율
- 판매가 합계 (¥/₩ 환산)
- 품절·역마진 상품 수

**등록 진행 배너**: 등록 진행 중일 때만 표시

**모니터링 탭 (항상 표시)**:
- 판매 중 (N)
- 변동 알림 중 (N)
- 판매 중지 (N)

**변동 알림 콜아웃 (항상 표시)**:
- 현재 등록 건수 / 최대 50건 (무료 플랜)
- "매일 오전 7시 가격·재고 자동 확인" 설명

**검색창**: 상품이 있을 때만 표시

**상품 테이블 (AllProductsTable)**:
- 컬럼: 체크박스 / 이미지 / 판매처(Qoo10) / 소싱처 로고+상품명 / 판매가 / 마진율 / 상품번호 / 등록일시 / 변동 알림 / 링크
- 상품명 호버: 한국어 원문 툴팁
- 빈 상태: ShoppingBag 아이콘 + "수집 목록에서 상품을 편집하고 Qoo10에 등록해 보세요."

**벌크 액션**:
- 판매 일시 중지 / 판매 재개
- 변동 알림 등록 / 해제
- 판매 종료 (삭제)

**변동 알림 상태 뱃지**:
- 정상 (초록 Shield)
- 가격 변동 (주황 TrendingDown)
- 역마진 (빨강 AlertTriangle)
- 품절 (회색 PackageX)
- 미등록 (회색 Minus)

### RG-02 판매 상품 상세 (`/registration/:resultId`)
- 개별 상품 상세 정보
- 모니터링 이력 확인

## UT 시나리오 설계 시 확인 포인트

1. **온보딩 완주율**: 3단계(기본정보 → Qoo10 연결 → 마진 설정)를 완료하지 못하고 이탈하는 지점은?
2. **수집 → 편집 흐름**: URL 입력 후 수집 완료 → 수집 목록 이동을 자연스럽게 따라가는가?
3. **편집 필요 태그 인지**: 상품 카드의 "편집 필요 · 상품명 / 옵션 / 상세설명" 태그를 보고 무엇을 해야 하는지 이해하는가?
4. **편집 탭 구조 파악**: 4개 탭(기본 정보 / 가격 / 썸네일 / 상세 이미지) 중 무엇부터 보는가?
5. **AI 번역 버튼 발견**: 상품명 / 옵션 / 상세설명 각각의 AI 버튼을 발견하고 사용하는가?
6. **Qoo10 등록 진입점**: 편집 완료 후 어디서 등록을 시도하는가 (상세 헤더 버튼 vs 목록 벌크 바)?
7. **판매 현황 이해**: 등록 후 "판매 중인 상품" 탭에서 대시보드와 상품 리스트를 보고 상태를 이해하는가?
8. **변동 알림 기능 인지**: 변동 알림 탭과 콜아웃을 보고 기능을 이해하고 등록하려 시도하는가?
