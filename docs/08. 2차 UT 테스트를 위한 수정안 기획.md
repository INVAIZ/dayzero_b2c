## UT 테스트 이전

### 준석님 피드백

- 온보딩과 실제 상품 페이지의 등록 페이지의 왼쪽 사이드 바 디자인 동일로 인해 사용자 혼란.
    - 온보딩 페이지의 모달 디자인 수정 필요.
- 수집하기 → 편집하기 → 판매 중인 상품 해당 흐름이 사용자가 이해하기 쉽지 않음. (첫 온보딩이 필요할 지 다른 방법이 필요할지)
    - 수집하기 탭에 수집된 상품 편집하기 탭이 있어야 할 지.
    - 판매 중인 상품과 판매 전 상품 구분을 어떻게 할 지.
- 편집하기에서 실제 상품의 어떤 부분을 편집해야 하는지 명확하지 않음.
    - 번역하기나 편집하기 버튼이 제대로 보이지 않음.
    - Qoo10에 등록하기 위해선 어떤 행동이 필요한지 명확하지 않음.
    - 상품을 호버하면 어떤 행동이 필요한지 보여주는 것도 필요하다. 현재 상품명 옵션 상세설명 태그만으로 사용자가 인식하기 쉽지 않다.

### UT 테스트 전 개인 피드백

- 각 설정 별로 하나씩 질문하는 것이 사용자가 이해하기 편하다.
    - 온보딩
        - 온보딩 내 마진 → 작업비 → 해외 배송비 → 실제 계산해볼래? 흐름.
    - 자동 수집 설정.
        - 어느 웹사이트에서 정보 긁어올래? → 카테고리는? → 몇 시에 긁어올래?

### 1. 탭 이동 — 개발 가이드 링크가 새 탭으로 열림

- 온보딩 중 개발 가이드 링크를 클릭하면 **새 탭으로 해당 페이지가 열림**
- 프로토타입 자체가 날아가는 건 아니지만, 사용자 입장에서 **갑자기 다른 화면으로 전환**되면서 흐름이 끊김
- 홍찬열 팀장님 테스트에서 발생. 다시 프로토타입 탭으로 돌아와서 진행함
- **심각도는 낮음.** 프로토타입이 사라지는 게 아니라 탭 전환만 발생하는 수준
- **수정:** 해당 링크를 비활성화하거나, 새 탭이 아닌 현재 탭 내에서 처리

### 2. 자동 수집 — UX 및 디자인 개선

**핵심: 사용자는 글을 안 읽는다. 텍스트가 아니라 디자인으로 알려줘야 한다.**

**문제 상황**

- 두 명 모두 자동 수집 설정 완료 후, 지금 바로 수집이 시작되는 줄 착각함
- 실제로는 "매일 오전 7시"에 실행되는 건데, 이걸 알려주는 화면 요소가 없음
- 홍찬열 팀장님: "글을 잘 안 읽게 됨" → 텍스트 안내에 의존하는 현재 설계의 한계를 직접 지적
- 손건우 주임님: 자동수집 탭으로 먼저 진입 → 설정 완료 → 아무 피드백 없음

**수정 방향**

- 설정 완료 시 토스트 또는 모달로 "자동 수집이 등록되었습니다. 다음 수집: 내일 오전 7시" 같은 **시각적 확인 피드백** 추가
- 텍스트 설명이 아닌 **상태 배지, 아이콘, 색상** 등 UI 요소로 동작 방식 전달
- 자동 수집 시간대 선택 드롭다운 추가 (두 명 모두 시간대 선택 니즈 언급)
    - 홍찬열 팀장님: "시간대를 선택하면 좋겠다"
    - 손건우 주임님: 주문 집중 시간(9~10시) 대비 7시 고정이면 품절 대응이 늦을 수 있다고 지적

### 3. 편집 화면 (ED-02) — 정보 구조 및 디자인 개선

**핵심: 사용자가 이 화면에서 "뭘 편집해야 하는지" 모른다. 어떤 필드가 번역이 필요한지, AI가 뭘 해주는지, 원본과 업로드 정보의 차이가 뭔지 전혀 안 보인다.**

**문제 ①: 어떤 필드를 편집해야 하는지 모르겠음**

- 준석님 사전 피드백: "편집하기에서 실제 상품의 어떤 부분을 편집해야 하는지 명확하지 않음", "번역하기나 편집하기 버튼이 제대로 보이지 않음"
- 손건우 주임님: "어떤 게 수정되지 않았는지, 어떤 걸 수정해야 하는지" 헷갈림
- **수정:** 각 필드에 편집 상태 표시 ("번역 필요", "AI 작성 가능", "수정됨", "원본 그대로" 등). 번역 안 된 필드는 시각적 강조 (경고 아이콘, 색상 등)

**문제 ②: 제목/옵션 등 번역 안 된 필드가 한국어 그대로인데, 원본인지 번역 전인지 구분 불가**

- 손건우 주임님: "한국어가 일본어로 자동으로 되는 줄 알았어요" → AI 자동 번역을 기대했는데 실제로는 버튼을 눌러야 함
- 홍찬열 팀장님: AI가 작성한 건지 원본인지 구분 안 됨
- **수정:** 한국어 원본 필드에 "한국어 원본 — 번역 필요" 라벨 명시. "번역하기" 버튼을 필드 바로 옆에 눈에 띄게 배치. 번역 완료 필드에는 "일본어 번역 완료" 상태 표시

**문제 ③: 상세 설명 빈칸 — AI가 뭘 하는 건지 모름**

- 홍찬열 팀장님: "상세 설명은 빈칸이었다. 헷갈렸다", "어디서 긁어오는 정보인지, AI가 이 상품 정보를 토대로 작성이 되었는지에 대한 설명이 없어"
- **수정:** 빈칸 상태일 때 "AI가 상품 정보를 기반으로 상세 설명을 작성합니다" 안내 + "AI 작성" CTA 버튼 배치. AI가 작성한 결과에는 "AI 생성" 태그 표시 (예측/실제 수집 태그 패턴과 동일)

**문제 ④: 수집 원본 정보와 Q10 업로드 정보를 비교할 수 없음**

- 손건우 주임님: "기존에 수집된 정보랑 Q10에 업로드 될 정보를 확인할 수 있으면 좋겠다", "원본 상품명 밑에 올릴 상품명 칸을 하나 더 만들어주시면 좋겠다"
- **수정:** 원본(수집 데이터)과 업로드용(편집 데이터)을 나란히 보여주는 구조. 원본 상품명은 읽기 전용, 업로드용은 편집 가능 필드로 분리

**문제 ⑤: 탭 이동 시 저장 불안**

- 손건우 주임님: "가끔 탭 옮기면 작업했던 정보가 사라지는 프로그램들이 있었던 것 같아서"
- **수정:** 화면에 "✓ 자동 저장됨" 상태 텍스트 표시

### 4. 변동 알림 — 알림 이후 바로 행동할 수 있게

**핵심: 알림만 오고 끝이면, 결국 Q10에 직접 들어가서 처리해야 한다. 알림에서 바로 액션이 이어져야 진짜 가치가 있다.**

**두 테스터 공통 발언**

- 손건우 주임님: "변경하고 재업로드 할 수 있는 기능이 있으면 좋을 것 같네요"
- 홍찬열 팀장님: "역마진이 나거나 품절이 났을 때 상품 일시정지가 자동으로 되는 시스템이면 훨씬 유용하다"

**수정 ①: 품절/역마진 분리 표시**

- 손건우 주임님: "품절은 나중에 재고 들어오면 다시 팔 수 있는데, 역마진은 아예 삭제하거나 해야 됨. 상황이 완전히 다르다"
- 변동 알림 화면에서 "품절" 탭과 "역마진" 탭을 분리 표시

**수정 ②: 품절 시 판매 일시정지 On/Off 버튼**

- 품절 알림 옆에 "판매 일시정지" 토글 버튼 추가
- 프로토타입에서는 버튼 UI + 확인 모달까지

**수정 ③: 역마진 시 가격 수정 버튼**

- 역마진 알림 옆에 "가격 수정" 버튼 추가
- 클릭 시 가격 편집 화면으로 이동 또는 인라인 수정

**수정 ④: 소싱 링크 바로가기**

- 손건우 주임님: "업로드 마켓 링크하고 소싱 링크 해서 바로 알 수 있는 링크 하나 있으면 좋겠다"
- 각 상품 알림에 Q10 링크 + 소싱처 링크 두 개를 바로 노출

**수정 ⑤: 변동 기록 (히스토리)**

- 홍찬열 팀장님: 변동 기록 기능의 유용성을 명확하게 체감
- 상품별로 가격/재고 변동 이력을 간단히 보여주는 영역 추가

### A. 온보딩 — 단계별 질문 형식으로 전환

현재 온보딩은 한 화면에 설정 항목들이 나열되어 있는 구조. 이걸 **한 번에 하나씩 질문하는 스텝 형식**으로 바꾼다.

- [ ]  **A-1. 온보딩 흐름을 스텝 형식으로 재설계**
    - 마진 설정 → 작업비 설정 → 해외 배송비 설정 → "실제 계산해볼래?" 순서
