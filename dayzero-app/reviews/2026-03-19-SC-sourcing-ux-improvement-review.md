상품 수집 화면 전반의 사용성을 개선했습니다. 자동 수집에 무료 플랜 제한(최대 10건) 표시를 추가하고, 불필요한 설명 박스를 제거하거나 조건부로만 보여주도록 변경했습니다. 직접 수집의 URL 입력 안내 문구를 더 직관적으로 개선하고, 입력 갯수 제한을 해제했습니다. 자동 수집 설정 화면의 시간 선택 드롭다운이 바깥을 클릭해도 닫히지 않던 문제도 수정했습니다.

## ➕ 계획 외 추가 사항

- **무료 플랜 콜아웃 (자동 수집)**: 기획 명세에 없지만 가격 정책 UT 검증을 위해 추가. 변동 알림과 동일한 패턴으로 일관성 확보. — **유지 권장**
- **URL 20개 제한 해제**: 명세서에는 "최대 20개"로 정의되어 있으나, 사용자 판단으로 제한 해제. — **유지 (UT 결과에 따라 재검토)**
- **스케줄 카드 문장 구조 변경**: "카테고리 상품의 ... 을 수집해요" → "카테고리의 ... 을 매일 HH:MM에 수집해요" 로 시간 정보를 카드 본문에 통합. — **유지 권장**

`UrlSourcingContent.tsx`는 전반적으로 하드코딩된 px 값이 많습니다 (`'16px'`, `'24px'`, `'12px'`, `'13px'`, `'15px'` 등). 리팩토링 시 일괄 토큰 교체 권장.

## 📝 의사결정 기록

### 자동 수집 콜아웃 조건부 표시
- **결정**: 스케줄 0건 → 설명 콜아웃, 1건 이상 → 무료 플랜 상태 콜아웃
- **고려한 대안**: 항상 설명 콜아웃 표시 / 항상 숨김
- **선택 이유**: UT에서 사용자가 콜아웃을 읽지 않는다는 관찰. 처음엔 필요하지만 익숙해지면 공간 낭비
- **트레이드오프**: 상태에 따라 다른 UI → 약간의 복잡도 증가, 하지만 사용자 경험은 개선

### 직접 수집 콜아웃 제거
- **결정**: 콜아웃 완전 제거, placeholder로 대체
- **고려한 대안**: 한 줄로 축소 / 빈 상태에서만 표시
- **선택 이유**: placeholder가 이미 "지원 쇼핑몰의 상품 URL만 붙여넣으면..." 으로 사용법 안내
- **트레이드오프**: 최초 사용자에게 설명이 부족할 수 있으나, 입력 필드 자체가 직관적

### URL 20개 제한 해제
- **결정**: 명세서의 20개 제한 해제
- **고려한 대안**: 제한 유지
- **선택 이유**: 사용자 판단 — UT에서 제한이 불필요하다고 판단
- **트레이드오프**: 대량 입력 시 성능 리스크 존재하나, UT 범위에서는 무관

### 무료 플랜 콜아웃 도입
- **결정**: 자동 수집에 "N건 / 최대 10건 · 무료 플랜" 상태 표시 추가
- **고려한 대안**: 표시 안 함 / 설정 추가 시 경고
- **선택 이유**: 변동 알림과 동일한 패턴으로 과금 구조의 일관성 확보. UT에서 가격 정책 반응 관찰 목적
- **트레이드오프**: 실제 제한 로직은 미구현 (UI만)
